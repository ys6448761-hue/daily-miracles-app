# Feedback Loop System

에이전트, 스킬, MCP 도구의 개선 사항을 기록하고 추적하는 시스템입니다.

## 목적

- 문제 발견 → 원인 분석 → 해결 → 검증의 사이클 구축
- 개선 이력 추적으로 동일 문제 재발 방지
- 버전별 변경사항 관리

## 폴더 구조

```
feedback/
├── README.md              # 이 파일
├── agents-feedback.md     # 에이전트별 개선 기록
├── skills-feedback.md     # 스킬별 개선 기록
├── mcp-feedback.md        # MCP 도구별 개선 기록
└── improvement-log.md     # 전체 개선 이력 (타임라인)
```

## 피드백 기록 방법

### 1. 문제 발견 시

```markdown
### [날짜] - [제목]
- **문제**: [발견된 문제 설명]
- **원인**: [원인 분석]
- **해결**: [해결 방법]
- **적용**: [수정한 파일 경로]
- **결과**: [개선 결과]
```

### 2. 예시

```markdown
### 2025-12-28 - Summarizer 출력 형식 불일치
- **문제**: 마크다운 표가 깨지는 현상
- **원인**: 파이프(|) 문자 이스케이프 누락
- **해결**: 출력 템플릿에 이스케이프 처리 추가
- **적용**: claude/agents/summarizer.md
- **결과**: 표 출력 정상화 확인
```

## 피드백 수집 경로

| 경로 | 설명 |
|------|------|
| 사용자 피드백 | 카카오채널, 이메일 문의 |
| 로그 분석 | 서버 에러 로그, API 응답 분석 |
| 테스트 결과 | 자동화 테스트 실패 케이스 |
| 팀 리뷰 | 정기 리뷰 미팅 논의 사항 |

## 우선순위 기준

| 등급 | 기준 | 대응 시간 |
|------|------|----------|
| P0 | 서비스 장애 | 즉시 |
| P1 | 핵심 기능 오류 | 24시간 내 |
| P2 | 사용성 개선 | 1주일 내 |
| P3 | 품질 향상 | 다음 스프린트 |

## 주간 리뷰

매주 일요일 피드백 정리:
1. 한 주간 수집된 피드백 검토
2. 해결된 항목 체크
3. 미해결 항목 우선순위 재조정
4. improvement-log.md 업데이트
