# 🧪 Aurora5 2주 도그푸딩 가이드 (GitHub/노션용)
**기간**: 2026-01-01 ~ 2026-01-14 (14일)
**목표**: "많이 만드는 것"이 아니라 **'진짜 쓰는 것만 남기는 것'**

---

## 1) 도그푸딩의 핵심 규칙 (꼭 지키기)
1) **매일 3분**만 기록한다 (길게 쓰면 3일 안에 망함)
2) 기록은 "예쁘게"가 아니라 **의사결정/실행 연결**이 목적
3) 카톡은 **🔴긴급만(YES)** — 리포트는 노션/대시보드/이메일
4) 2주 후 확장(Bridge/자동화)은 **데이터로 결정**한다

---

## 2) 폴더/파일 규칙
**추천 구조:**
```
.claude/dogfooding/
├── README.md
├── DAILY_TEMPLATE.md
├── WEEKLY_TEMPLATE.md
├── 2026-01-01_day1.md
├── 2026-01-02_day2.md
└── ...
```

**파일명 규칙:**
- 일일: `YYYY-MM-DD_day{n}.md`
- 주간: `WEEK{n}_YYYY-MM-DD_to_YYYY-MM-DD.md` (선택)

---

## 3) 매일 루틴 (3분)
1) **아침 1장 리포트**를 보고 Top5를 채운다 (숫자 + 1줄 코멘트)
2) **오늘 의사결정 1개**를 반드시 적는다
3) 실행 여부(완료/진행/보류)를 체크한다
4) 🔴 알림이 있으면 오탐/미탐 여부만 기록한다
5) 오늘 실제로 쓴 도구 Top3만 적는다
6) 내일 개선 1개만 적는다

---

## 4) 무엇을 성공으로 볼 것인가 (Go/No-Go)

### Go (확장 진행)
- 리포트 확인률 **70% 이상** (예: 10/14일 이상)
- 의사결정 기록 **8회 이상**, 실행 연결 **5회 이상**
- 🔴 오탐/미탐이 "원인 파악 + 룰 개선"으로 관리 가능
- Top5 도구가 명확히 드러남("진짜 쓴 것")

### No-Go (단순화/정리부터)
- 데이터 부정확/지표 불신
- 알림 피로(오탐 많음)로 🔴를 무시하게 됨
- 도구는 많지만 실제 사용이 거의 없음

---

## 5) 2주 후 산출물 (반드시 남길 것)
- 주간 리포트 2개(Week1, Week2)
- "사용률 Top5 도구" 확정
- "쓸모없는 지표/알림(삭제 후보)" 리스트
- 🔴 오탐/미탐 통계 + 룰 개선안 1~2개
- 다음 2주 우선순위 3개(딱 3개)

---

## 6) 운영 팁 (실패 방지)
- 기록은 "감상"이 아니라 **숫자/행동** 중심
- 템플릿을 지키면, 2주 후 자동으로 **의사결정 시스템이 남는다**
- 뭔가 불편하면 "내일 개선 1개"에 적고, 한 번에 다 고치려 하지 말기

---

## 7) 커밋 룰 (선택)
- 매일 업무 종료 시 1회 커밋 권장
- 커밋 메시지 예:
  - `dogfooding: day3 log`
  - `dogfooding: week1 report`
