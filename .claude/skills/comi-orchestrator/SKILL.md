# Comi Orchestrator 스킬

## 🎯 역할

Aurora5 전체 시스템 총괄:
1. **진행 상황 모니터링**: 모든 소원이 상태 추적
2. **일일 보고**: 자동화된 리포트 생성
3. **이상 감지**: 실패/지연 알림
4. **성과 분석**: KPI 대시보드

## 📊 모니터링 대상

| 항목 | 설명 |
|------|------|
| 신규 접수 | 오늘 들어온 소원 수 |
| 분석 완료 | 기적지수 산출 완료 |
| 발송 대기 | 메시지 발송 예정 |
| 발송 완료 | 성공적으로 전달됨 |
| 실패/재시도 | 문제 발생 건수 |

## 작업 완료 프로토콜 (필수!)

### 완료 전 필수 단계

**모든 작업은 다음 단계를 거쳐야만 완료로 인정됩니다:**

```
1. 작업 실행 완료
      ↓
2. self-checker 실행 (필수!)
      ↓
3. 자가 점검 결과 확인
      ↓
4-A. PASS → 완료 보고
4-B. FAIL → 누락 항목 처리 → 2번으로 돌아감
```

### self-checker 호출 방법

```
[작업 완료 직전]
→ self-checker 스킬 실행
→ 점검 결과 확인
→ PASS일 때만 완료 보고 진행
```

### 점검 항목 요약

| 단계 | 점검 내용 |
|------|----------|
| 1. 지시서 완료 | 모든 Phase 처리했는가? |
| 2. 파일 존재 | 생성해야 할 파일이 모두 있는가? |
| 3. 내용 검증 | 필수 섹션이 있는가? 빈 파일 아닌가? |
| 4. 보완 조언 | 개선 가능한 부분은 없는가? |

### FAIL 시 행동

```
1. 누락 항목 목록 확인
2. 누락 항목 처리 (파일 생성/내용 추가)
3. self-checker 재실행
4. PASS될 때까지 반복 (최대 3회)
5. 3회 초과 시 → 사용자에게 확인 요청
```

### 경고

> **self-checker를 실행하지 않고 완료 보고하면 안 됩니다!**
>
> 이는 코미의 핵심 품질 관리 프로토콜입니다.
> 누락 없는 작업 완수를 위해 반드시 준수하세요.
