# Blog Writer 스킬

> 하루하루의 기적 블로그 마케팅 자동화
> 루미(Data Analyst) 전략 기반 v2.0

---

## 트리거

```
"블로그 써줘: [주제]"
"/blog [주제]"
"블로그 주제 뽑아줘"        ← 주제 생성 모드
"이번주 블로그 계획 짜줘"   ← 7일 큐 생성
```

---

## 입력

| 항목 | 필수 | 예시 |
|------|------|------|
| 주제 | O | 새해 소원 이루는 법 |
| 플랫폼 | X | 네이버, 티스토리, 브런치 |
| 톤 | X | 따뜻한, 전문적, 친근한 |
| 글자수 | X | 1500자 (기본값) |

---

## 주제 생성 공식 (루미 전략)

### 공식 A: 문제해결형 (검색량 높음)
```
[상황/증상] + [기간/마감] + 어떻게 해야 하나요?
```
- "이직이 너무 무서운데 어떻게 해야 하나요?"
- "관계가 멀어진 느낌, 다시 회복하려면?"

### 공식 B: 소원실현형 (전환 연결)
```
[소원] + 현실적으로 + 30일 안에 + 가능한가요?
```
- "30일 안에 루틴 만들기 현실적으로 가능할까요?"
- "30일 안에 자존감 회복, 뭘 하면 좋을까요?"

### 공식 C: 여수항해형 (브랜딩)
```
[여수/여행] + [힐링/새출발/관계] + 추천 코스
```
- "여수 1박2일 혼자 힐링 코스 추천"
- "여수에서 새해에 마음 정리하는 여행 코스"

## 로테이션 규칙 (1일 1포스트)

| 요일 | 카테고리 | 목적 |
|------|----------|------|
| 월/수/금 | 문제해결 (problem) | 검색 유입 |
| 화/목 | 소원실현 (wish) | 전환 연결 |
| 토/일 | 여수항해 (yeosu) | 브랜딩 |

## 카테고리별 CTA 연결

| 카테고리 | CTA | 링크 |
|----------|-----|------|
| problem | "문제해결 상담받기 →" | /problem-solving |
| wish | "나의 소원 등록하기 →" | /wish-form |
| yeosu | "여수항해 상담받기 →" | /wish-voyage |

## 파이프라인

```
[주제 입력]
     ↓
[1. 키워드 분석]
  - 메인 키워드 추출
  - 연관 키워드 5개
  - 검색 의도 파악
     ↓
[2. 구조 설계]
  - 제목 3개 후보
  - 소제목 구성
  - 훅(도입부) 설계
     ↓
[3. 본문 작성]
  - 서비스 연결 자연스럽게
  - CTA 삽입 위치 결정
  - 이모지 적절히 활용
     ↓
[4. 이미지 제안]
  - 소원그림 스타일 프롬프트
  - 또는 기존 이미지 추천
     ↓
[5. SEO 최적화]
  - 메타 설명
  - 해시태그 10개
  - 내부 링크 제안
     ↓
[출력: 발행용 마크다운]
```

## 톤앤매너 가이드

### 기본 톤: 따뜻한 희망

```
- "~해보세요" (권유형)
- "함께 ~해요" (동행형)
- 소원이 → 독자 호칭
- 기적, 희망, 응원 키워드 자연스럽게
```

### 금지 표현

```
- 과장된 약속 ("100% 이루어집니다")
- 부정적 표현 ("실패하면", "안 되면")
- 경쟁사 언급
```

## 플랫폼별 조정

| 플랫폼 | 글자수 | 특징 |
|--------|--------|------|
| 네이버 | 1500~2000 | 키워드 반복, 이미지 多 |
