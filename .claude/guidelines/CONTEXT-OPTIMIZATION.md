# 컨텍스트 최적화 가이드라인

## 원칙
> "컨텍스트는 금이다. 낭비하지 마라."

## 분리 기준

| 작업 유형 | 처리 방식 | 이유 |
|----------|----------|------|
| 지능 필요 | 에이전트 | AI 판단 필요 |
| 반복 작업 | 스크립트 | 정확성 보장 |
| 외부 연동 | MCP | 컨텍스트 절약 |
| 데이터 추출 | 스크립트 | 대량 처리 |
| 파일 생성 | 스크립트 | 정확성 보장 |

## 금지 사항

- 대량 데이터를 컨텍스트에 직접 로드
- 반복적인 파일 생성을 에이전트가 직접
- 외부 API 호출 결과 전체를 컨텍스트에
- 로그 전체를 컨텍스트에 포함

## 권장 사항

- 핵심 정보만 컨텍스트에
- 무거운 처리는 스크립트/MCP로
- 결과 요약만 에이전트에게 전달
- 필요한 데이터만 선별적으로 로드

## 예시

```
나쁜 예:
"데이터베이스의 모든 사용자 정보를 분석해줘"
→ 10만 건 데이터가 컨텍스트에!

좋은 예:
"데이터베이스에서 최근 7일 신규 사용자 통계만 조회해서 분석해줘"
→ 요약 데이터만 컨텍스트에
```
