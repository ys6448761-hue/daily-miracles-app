## 1. 요청 카드 템플릿 (필수 6필드)

모든 작업 요청은 아래 형식으로 제출해주세요.
**6필드 미충족 시 접수 불가**

```
📋 [요청 카드]

📌 목적(왜):
   → 한 줄로 "왜 필요한지"

🎯 결과물(무엇):
   → 예) "GA4 로더 삽입 PR", "슬랙 공지문", "Airtable 스키마"

⚠️ 영향도: 🔴 긴급 / 🟡 중요 / 🟢 개선

📅 마감:
   → YYYY-MM-DD HH:MM

👤 담당(희망):
   → 코미 / 루미 / 재미 / 여의보주 / Claude Code

✅ 검증 기준(완료 정의):
   → 예) "DebugView에 P0 이벤트 3개 수신"
```

### 예시

```
📋 [요청 카드]

📌 목적: GA4 전환 추적이 안 되어 마케팅 성과 측정 불가
🎯 결과물: index.html 외 5개 파일에 GA4 로더 삽입 + 커밋
⚠️ 영향도: 🔴 긴급
📅 마감: 2026-01-16 12:00
👤 담당: Claude Code
✅ 검증: GA4 DebugView에서 itinerary_builder_submit 이벤트 수신 확인
```

## 2. 라우팅 매트릭스 (담당자 자동 배정)

| 키워드/카테고리 | 담당자 | 비고 |
|----------------|--------|------|
| 코드, 배포, DB, API, 마이그레이션, PR | **Claude Code** | 기술 구현 |
| 지표, 분석, GA4 설계, 임계값, 대시보드 | **루미** | 데이터 분석 |

| 카피, 디자인, 시각자료, SNS 콘텐츠, 스냅샷 | **재미** | CRO 창의 |
| 철학, 메시지 톤, 후기 검수, 브랜드 감성 | **여의보주** | 품질 검수 |
| 결제, CS, 고객 응대, 예산, 계약 | **푸르미르** | CEO 최종 |

- 주 담당 + 협업 담당 명시
- 예: "GA4 이벤트 설계(루미) + 코드 삽입(Claude Code)"

## 3. 우선순위 SLA (승인 없이 실행 가능 범위)

| 등급 | 정의 | 처리 방식 | 푸르미르 개입 |
|------|------|----------|--------------|
| **P0** | 긴급 (매출/장애/서비스 중단) | **즉시 착수**, 담당자 자율 실행 | ❌ 불필요 |
| **P1** | 중요 (이번 주 내) | 코미가 일정 확정 후 실행 | ❌ 불필요 |
| **P2** | 개선 (다음 주) | 주 1회 배치 처리 | ❌ 불필요 |
| **P3** | 아이디어/보류 | 백로그 저장 | ❌ 불필요 |

### 푸르미르 개입이 필요한 2가지 경우만
1. **DEC(결정)가 필요한 의사결정** - 정책/원칙 변경
2. **예산/리스크/브랜드 방향** - 비용 발생, 외부 영향

## 4. 문서 3종 분리 규칙

| 문서 유형 | 목적 | 보는 사람 | 예시 |
|----------|------|----------|------|
| **DEC** | 바뀌면 안 되는 결정 | 푸르미르 | DEC-006: Ops Agent 운영 체계 |
| **Task** | 해야 할 일 | 담당자 | "GA4 로더 삽입" 요청 카드 |
| **Snapshot** | 현황 공유 | 전체 팀 | 프로젝트 스냅샷 1페이지 |

## 5. 주 2회 트리아지 (분류 회의)

| 항목 | 내용 |
|------|------|
| **일시** | 화/금 15분 (또는 매일 10분) |
| **참석** | 코미 (필수), 담당자 (선택) |
| **어젠다** | 백로그 → P0/P1 선별 → 이번 스프린트 이동 |
| **산출물** | 코미가 5줄 고정 메시지 공유 |

### 트리아지 후 공유 포맷 (코미용)

```
📊 [이번 스프린트] 2026-01-15 화

🔴 P0 (오늘):
  • GA4 로더 삽입 @Claude Code

🟡 P1 (이번 주):
  • 10회 검증 완료 @재미
  • Airtable 테이블 생성 @루미

🟢 P2 (다음 주):
  • 피드백 페이지 복구
  • 후기 수집 자동화

📌 DEC 대기: 없음
```

## 6. 슬랙 공지용 요약 (코미가 복사해서 사용)

```
📢 [Aurora 5 팀 운영 규칙] 적용 시작

1️⃣ 모든 요청은 "요청 카드 6필드"로만 접수
   (목적/결과물/영향도/마감/담당/검증기준)

2️⃣ 담당자 자동 배정:
   • 코드/배포 → Claude Code
   • 분석/지표 → 루미
   • 공지/일정 → 코미
   • 카피/디자인 → 재미
   • 톤/검수 → 여의보주

3️⃣ P0(긴급)은 승인 없이 먼저 처리 가능
   푸르미르 개입: DEC 필요 or 예산/리스크만

4️⃣ 백로그 운영: 코미 담당
   트리아지: 화/금 15분

5️⃣ 문서 3종: DEC(결정) / Task(실행) / Snapshot(공유)

🔗 상세: .claude/TEAM_OPS_GUIDE.md
```
