# 블로그 외부 검증 + Slack 승인 카드 스펙

---

## ✅ 외부 전문가 검증 체크리스트 (10문항)

**목적:** "사람이 쓴 글처럼 자연스럽고, 과장/근거 불명 표현 없이, SEO 기본기를 갖춘 글"만 통과

### 0) 판정 규칙

| 판정 | 조건 |
|------|------|
| **Approve** | ✅ 8개 이상 + 🔴 레드플래그 0개 |
| **Needs Fix** | ✅ 6~7개 또는 🔴 1개 |
| **Reject** | ✅ 5개 이하 또는 🔴 2개 이상 |

### 1) 체크 항목 10개

1. **첫 5줄이 끌리나요?** (공감/문제 제기 명확)
2. **핵심 메시지 1문장**이 중간/초반에 보이나요?
3. **구조가 읽기 쉬운가요?** (H2 3개 내외, 문단 과밀 X)
4. **브랜드 톤**이 따뜻/희망/친근인가요? (차갑거나 과장된 설교 X)
5. **'사람의 경험/감정 1컷'**이 들어가 있나요? (2~4문장)
6. **단정/과장 표현**이 과하지 않나요? ("증명/무조건/100%/확실히" 등)
7. **근거 없는 수치/데이터**가 없나요? (있다면 출처가 명시되어 있나요?)
8. **금지어/오해 유발**이 없나요? (사주/운세/궁합/부적/초능력 느낌)
9. **CTA가 자연스럽나요?** (강압/불안 조장 X, 1개로 깔끔)
10. **SEO 기본 패키지**가 갖춰졌나요? (메타 설명 적정, 태그 2~3, 내부링크 2개)

### 🔴 레드플래그 (있으면 무조건 Fix 이상)

- "5,000명 데이터/달성률 87%/효과 +34%"처럼 **증빙 없는 숫자**
- "뇌과학이 증명/과학적으로 확실" 같은 **강한 단정**
- 독자가 불편해질 수 있는 **공격/비난/불안 자극/허위 약속**
- 금지어(사주/운세/궁합 등) 직접 언급

### 외부 리뷰어 피드백 템플릿 (회신용)

```
판정: Approve / Needs Fix / Reject

수정 1순위 (최대 3개):
1)
2)
3)

좋은 점 (1~2개):


추가 제안 (선택):

```

## ✅ Slack 승인 카드 스펙 (코드 구현용)

### 1) 메시지 구조

**제목:**
```
📝 블로그 글 검수 요청: {title}
```

**본문:**
```
• 주제: {topic}
• 키워드: {keywords}
• 발행 방식: 예약({schedule_at}) 또는 초안
• 상태: {Status} / 신호등: {TrafficLight}

📎 미리보기: {preview_url}

📋 QA 요약:
• 리스크: {risk_flags}
• SEO 체크: {seo_pass_fail}
• 사람 1컷: {purmir_story_cut_present}
```

### 2) 버튼 3개

| 버튼 | 이모지 | 라벨 |
|------|--------|------|
| 승인 | ✅ | Approve |
| 수정요청 | 🛠 | Needs Fix |
| 거부 | ⛔ | Reject |

### 3) 버튼 클릭 시 상태 전이 (Airtable)

#### ✅ Approve 클릭
```
Review_Status = APPROVED
Status = APPROVED
TrafficLight = 🟢 green
→ 다음: Publisher (Ghost 예약/발행) 실행
```

#### 🛠 Needs Fix 클릭
```
Review_Status = NEEDS_FIX
Status = NEEDS_HUMAN
TrafficLight = 🟡 yellow
→ 다음: Writer 재작성 요청 (수정 포인트 3개만 받아 재생성)
```

#### ⛔ Reject 클릭
```
Review_Status = REJECTED
Status = BLOCKED
TrafficLight = 🔴 red
→ 다음: 발행 중지 + 코미/푸르미르 에스컬레이션
```

### 4) Fix 요청 시 추가 입력 (핑퐁 방지)

Needs Fix 클릭 시 **수정 포인트 최대 3줄**만 입력:
- `fix_1`:
- `fix_2`:
- `fix_3`:

→ 이 3줄만 프롬프트에 주입해서 **"재작성 1회"**로 끝내기
