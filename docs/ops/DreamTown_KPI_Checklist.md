## 일일 점검 항목

### ✅ KPI-1: 오늘 퍼널 전환율

실행: `[KPI-1]` 쿼리

| 지표 | 정상 기준 | 비고 |
|------|-----------|------|
| scene_view 수 | 1 이상 | 당일 데이터 없으면 수집 미시작 |
| sv→sac 전환율 | 측정 중 | 기준선 수집 단계 |
| tov→ca 전환율 | 측정 중 | 상품 클릭률 |
| 최종 전환율 | 측정 중 | A/B 종료 후 승자 기준 확정 |

---

### ✅ KPI-2: A/B 실험 결과 (coupon_test_1)

실행: `[KPI-2]` 쿼리

| 체크 항목 | 판단 기준 |
|-----------|-----------|
| variant A 수집 수 | 10 이상이면 유효 |
| variant B 수집 수 | 10 이상이면 유효 |
| 전환율 차이 | 3%p 이상 차이 시 → 우세 variant 확인 |
| (없음) 행 | 실험 외 진입 — 무시 가능 |

**A/B 결과 판독 기준**

```
A 전환율 > B 전환율 + 3%p → A 승
B 전환율 > A 전환율 + 3%p → B 승
차이 3%p 미만 → 데이터 추가 수집 필요
```

## 이벤트 검증 체크리스트

```
실행: [검증-1] ~ [검증-3] 쿼리
```

| 항목 | 확인 방법 | 정상 |
|------|-----------|------|
| 허용외 이벤트 없음 | [검증-1] status 컬럼 | '⚠️ 허용외' 0행 |
| travel_offer_view 순서 정상 | [검증-2] 결과 | 0행 |
| conversion_action 중복 없음 | [검증-3] 결과 | 0행 |
| coupon_open 미발생 | [레거시] 쿼리 | count = 0 |

---

## 퍼널 흐름 (정상 경로)

```
wish_start
  → scene_view
    → scene_action_click   ← 선택 버튼 클릭
      → travel_offer_view  ← 상품 카드 노출 (감정 peak 이후만)
        → conversion_action
```

---

## 레거시 현황

| 이벤트 | 상태 | 조치 |
|--------|------|------|
| `coupon_open` | 허용 목록 유지 / 실제 발생 없음 | 현행 유지 (삭제 불요) |

> `coupon_open`이 실제 발생한다면 → 구 코드 잔존 가능. 소스 추적 후 보고.
