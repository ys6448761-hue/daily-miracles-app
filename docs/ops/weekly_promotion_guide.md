# 주간 지식카드 승격 가이드

> RAW → Knowledge Card 승격 운영 매뉴얼

---

## 운영 규칙

| 항목 | 규칙 |
|------|------|
| **빈도** | 주 1회 (일요일) |
| **승격 개수** | 최대 5개 (고정) |
| **담당** | 푸르미르님 선별 → 루미 카드화 → 코미 저장 |

---

## 승격 기준

### 필수 조건 (모두 충족)

1. **재사용 가능성** - 같은 상황에서 다시 참조할 가치가 있음
2. **범용성** - 특정 1회성 작업이 아닌, 여러 상황에 적용 가능
3. **팀 공유 가치** - 개인 메모가 아닌, 팀 전체에 유용

### 가산 조건 (우선순위 높음)

- [ ] 실제 적용 후 효과가 검증됨 → **#검증됨** 태그 추가
- [ ] 자주 검색되거나 언급됨
- [ ] 의사결정의 근거가 됨
- [ ] 반복 질문에 대한 답변

### 제외 조건 (승격 불가)

- 개인 일정/메모
- 민감 정보 포함 (개인정보, 금융정보)
- 일회성 버그 수정 기록
- 이미 승격된 내용과 중복

## 승격 프로세스

### Step 1: 선별 (푸르미르님, 일요일 오전)

```
1. RAW_INBOX 또는 docs/raw/ 폴더 전체 훑어보기
2. 승격 기준에 맞는 문서 5개 이내 선별
3. 선별 목록을 Slack #knowledge 채널에 공유
```

**선별 템플릿:**
```
📋 이번 주 승격 후보 (YYYY-MM-DD)

1. [파일명] - 선별 이유
2. [파일명] - 선별 이유
3. [파일명] - 선별 이유
...

총 N개 / 최대 5개
```

### Step 2: 카드화 (루미, 일요일 오후)

```
1. 선별된 RAW 문서 읽기
2. docs/templates/knowledge_card_template.md 복사
3. 템플릿에 맞게 내용 정리
4. 태그 부여 (docs/standards/tag_taxonomy.md 참조)
5. 파일명: KC-YYYY-MMDD-###.md
```

**파일 저장 위치:**
```
docs/knowledge/
├── 2026-01/
│   ├── KC-2026-0122-001.md
│   ├── KC-2026-0122-002.md
│   └── ...
└── 2026-02/
    └── ...
```

### Step 3: 저장 및 검증 (코미, 일요일 저녁)

```
1. docs/knowledge/ 폴더에 파일 저장
2. Git commit & push
3. 검색 테스트 (키워드로 찾아지는지 확인)
4. Slack #knowledge 채널에 완료 보고
```

**완료 보고 템플릿:**
```
✅ 주간 지식카드 승격 완료 (YYYY-MM-DD)

신규 카드:
- KC-XXXX-XXXX-001: [제목] #태그1 #태그2
- KC-XXXX-XXXX-002: [제목] #태그1
...

총 N개 승격 완료
누적 카드: XX개
```

## 주의사항

### 운영 부담 방지

| 규칙 | 이유 |
|------|------|
| **주 5개 초과 금지** | 품질 유지, 운영 지속 가능성 |
| **강제 승격 없음** | 후보가 없으면 0개도 OK |
| **RAW도 검색 가능** | 승격은 "하이라이트"일 뿐, RAW도 가치 있음 |

### 품질 유지

- 요약은 **50자 이내**로 작성
- 핵심 인사이트는 **3줄 이내**
- 적용 가능 상황은 **구체적으로** 작성
- 관련 카드 연결은 **실제로 관련 있을 때만**

### 지속 가능성

- 일요일 놓쳤으면 **다음 주로 이월** (몰아서 10개 금지)
- 담당자 부재 시 **푸르미르님이 직접** 또는 **스킵**
- 복잡하게 느껴지면 **단순화** (가이드 수정 요청)

## 검색 활용

### 태그 기반 검색

```bash
# 특정 태그의 모든 카드
grep -r "#전략" docs/knowledge/

# 검증된 콘텐츠만
grep -r "#검증됨" docs/knowledge/
```

### 키워드 검색

```bash
# 특정 키워드 포함 카드
grep -ri "소원이" docs/knowledge/

# 제목에서 검색
grep -r "^# " docs/knowledge/ | grep "마케팅"
```

### VS Code에서 검색

1. `Ctrl + Shift + F` (전체 검색)
2. 검색 범위: `docs/knowledge/`
3. 파일 유형: `*.md`

## FAQ

### Q: RAW와 Knowledge Card의 차이?

| 구분 | RAW | Knowledge Card |
|------|-----|----------------|
| 형식 | 자유 | 표준 템플릿 |
| 용도 | 원본 보관 | 빠른 참조 |
| 검색 | 전문 검색 | 태그 + 구조화 검색 |
| 관리 | 축적 | 큐레이션 |

### Q: 승격 안 된 RAW는 삭제?

**아니요.** RAW는 영구 보관합니다. 승격은 "하이라이트"일 뿐입니다.

### Q: 5개 넘는 좋은 후보가 있으면?

다음 주로 이월합니다. 5개 제한은 품질과 지속 가능성을 위한 것입니다.

### Q: 이미 승격된 내용이 업데이트되면?

기존 카드에 **변경 이력**을 추가합니다. 새 카드를 만들지 않습니다.
