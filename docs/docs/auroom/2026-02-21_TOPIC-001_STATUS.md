맞아요 푸르미르님 — **“warroom”은 제가 편의상 붙인 이름**이고, 푸르미르님이 말씀하신 것처럼 **“auroom(아우룸)”으로 쓰는 게 더 일관**돼요.

핵심은 폴더 이름이 아니라 **“한 주제 = 정본 1장(STATUS) = 팀이 보는 같은 화면”** 구조예요.

---

## 1) warroom vs auroom 차이

* **warroom**: 일반적으로 “전쟁방/상황실” 느낌(급박한 대응)
* **auroom(아우룸)**: 우리 프로젝트 컨셉에 맞는 “회의센터/오케스트레이션 룸” 느낌

그래서 **우리는 앞으로 `auroom`으로 통일**하는 게 좋아요.

---

## 2) GitHub 폴더명 추천 (통일안)

저는 이렇게 추천합니다:

* `docs/auroom/`  ✅ (회의센터 정본)
* (선택) `docs/auroom/topics/`  ✅ (주제별 폴더를 더 엄격히 나누고 싶으면)

초보자 운영에선 **`docs/auroom/` 하나면 충분**해요.

---

## 3) 이미 docs/warroom을 만들었으면 어떻게 하지?

둘 중 편한 걸 택하면 됩니다.

### ✅ A안: 그냥 계속 warroom 써도 됨 (가장 쉬움)

* 지금 만든 걸 유지
* 이름만 “아우룸 회의센터”라고 부르면 됨

### ✅ B안: auroom으로 바꾸기 (추천)

GitHub 웹에서 이렇게 하면 돼요:

1. `docs/warroom/` 들어가기
2. 파일 하나 클릭
3. 연필(✏️) Edit 버튼
4. 위쪽 파일명에서
   `docs/warroom/...` 를 `docs/auroom/...` 으로 바꾸기
5. Commit

이걸 파일 1~2개만 옮기면, 폴더가 자동으로 `docs/auroom/` 생겨요.
(폴더는 “파일 경로”로 생성되니까요.)

> ⚠️ 폴더 단위 “rename”은 GitHub 웹에서 한번에 하기 애매할 수 있어서
> **처음엔 파일 1개만 auroom으로 만들고, 새로 그쪽에 쌓는 방식**이 제일 안전합니다.

