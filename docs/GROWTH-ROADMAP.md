## Phase 2: 고도화 (1월 ~ 2월)

### 목표
자동화 강화 + 운영 효율화

### 주요 작업

#### 1. 자동 보고 시스템
- 일일 리포트 자동 생성
- 코미/재미에게 자동 발송

#### 2. 배치 처리
- 7일 응원 메시지 일괄 발송
- 로드맵 일괄 생성

#### 3. 모니터링 대시보드
- 소원이 현황 한눈에
- 신호등 상태 실시간
- **재미의 에어테이블 중앙관제센터 연동**

#### 4. 피드백 루프
- 소원이 반응 수집
- 메시지 효과 분석

### 검증 포인트
- "10명, 50명, 100명도 감당 가능한가?"
- "어디서 병목이 생기나?"

---

## Phase 3: 소원착지 (3월 ~)

### 목표
100~200개 멀티 프로젝트 운영

### 주요 작업

#### 1. 프로젝트 팩토리
```
소원이 등록 → 프로젝트 자동 생성 → 템플릿 기반 초기화
```

#### 2. 중앙 관제탑
- 100개 프로젝트 한눈에
- 지연/위험 자동 감지
- 재미 에어테이블과 연동

#### 3. 크로스 프로젝트 학습
- 성공 패턴 축적
- 신규 소원이에게 자동 추천

#### 4. 폴더 구조
```
sowon-landing/
├── .aurora/
│   ├── central.md
│   ├── templates/
│   └── patterns/
├── projects/
│   ├── active/
│   ├── completed/
│   └── archived/
└── reports/
```

## 최수민님 방식 vs 확장 방식

| 구분 | 최수민님 (Phase 1-2) | 확장 (Phase 3) |
|------|---------------------|----------------|
| **초점** | 개인 생산성 | 조직 생산성 |
| **규모** | 1인 1프로젝트 | 팀 + N프로젝트 |
| **학습** | 세션 내 | 프로젝트 간 전이 |
| **관리** | 개별 | 중앙 집중 |
