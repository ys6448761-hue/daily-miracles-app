# 태그 체계 v1.0

> 하루하루의 기적 지식 관리 표준 태그 체계

---

## 필수 태그 (택 1)

모든 문서는 아래 6개 중 **하나의 필수 태그**를 가져야 합니다.

| 태그 | 코드 | 설명 | 예시 |
|------|------|------|------|
| **#전략** | `STRATEGY` | 방향성, 의사결정, 비전 | 마케팅 전략, 제품 로드맵, 팀 방향 |
| **#기술** | `TECH` | 개발, 시스템, API, 코드 | 버그 수정, 배포 방법, API 연동 |
| **#운영** | `OPS` | 프로세스, 루틴, 워크플로우 | 베타 운영, 팀 관리, 일일 체크 |
| **#콘텐츠** | `CONTENT` | 메시지, 디자인, 카피, 영상 | 응원 메시지, 광고 소재, 대본 |
| **#분석** | `ANALYSIS` | 데이터, 인사이트, 리서치 | 경쟁사 분석, 시장 조사, 지표 분석 |
| **#고객** | `CUSTOMER` | 소원이, 피드백, VOC | 사용자 반응, 고객 문의, 리뷰 |

---

## 선택 태그 (0-2개)

필수 태그 외에 **최대 2개**까지 선택 태그를 추가할 수 있습니다.

| 태그 | 코드 | 설명 | 사용 조건 |
|------|------|------|----------|
| **#긴급** | `URGENT` | 즉시 참조/실행 필요 | 24시간 내 액션 필요 시 |
| **#검증됨** | `VERIFIED` | 실제 적용 후 효과 확인 | 실행 결과가 증명된 경우 |

---

## 태그 규칙

### 1. 태그 개수
- **최소**: 1개 (필수 태그)
- **최대**: 3개 (필수 1 + 선택 2)

### 2. 태그 형식
```
#한글태그 또는 영문코드
예: #전략, #기술, STRATEGY, TECH
```

### 3. 금지 사항
- 새로운 태그 임의 생성 금지
- 필수 태그 없이 선택 태그만 사용 금지
- 4개 이상 태그 사용 금지

## doc_type ↔ 태그 매핑

기존 시스템 doc_type과의 호환성:

| doc_type | 기본 태그 | 비고 |
|----------|----------|------|
| `SYSTEM` | #기술 | 배포, 에러, API 관련 |
| `DECISION` | #전략 | 결정, 정책, 승인 |
| `CONTENT` | #콘텐츠 | 영상, 대본, 메시지 |
| `NOTE` | 내용 분석 후 결정 | LLM 판단 |

## 태그 선택 가이드

### 빠른 판단 흐름도

```
문서가 코드/시스템 관련?
├─ YES → #기술
└─ NO → 의사결정/방향 관련?
         ├─ YES → #전략
         └─ NO → 고객/소원이 관련?
                  ├─ YES → #고객
                  └─ NO → 데이터/분석 관련?
                           ├─ YES → #분석
                           └─ NO → 메시지/콘텐츠 관련?
                                    ├─ YES → #콘텐츠
                                    └─ NO → #운영
```

### 선택 태그 기준

**#긴급** 추가 조건:
- 오늘/내일 안에 실행해야 함
- 지연 시 비즈니스 영향 있음
- RED 신호등 상황

**#검증됨** 추가 조건:
- 실제 프로덕션에서 테스트 완료
- 성과 지표로 효과 확인됨
- 재사용 시 동일 결과 기대 가능

## 예시

### 예시 1: 배포 관련 기록
```yaml
tags: [#기술, #긴급]
# 필수: 시스템/코드 관련 → #기술
# 선택: 즉시 적용 필요 → #긴급
```

### 예시 2: 마케팅 전략 회의
```yaml
tags: [#전략]
# 필수: 방향성 결정 → #전략
# 선택: 해당 없음
```

### 예시 3: 소원이 피드백 분석
```yaml
tags: [#고객, #분석]
# 필수: 소원이 관련 → #고객
# 선택: 데이터 분석 포함 → #분석
```

### 예시 4: 검증된 응원 메시지 템플릿
```yaml
tags: [#콘텐츠, #검증됨]
# 필수: 메시지 템플릿 → #콘텐츠
# 선택: 실제 효과 확인됨 → #검증됨
```
