# 최수민님 방식 완전 정리

> "AI는 추론, 반복은 코드로!"
> "내가 쓸 제품을 만들어라" (도그푸딩)

---

## 1. 핵심 철학

### 1인 유니콘의 조건
- AI를 **추론 엔진**으로 활용
- 반복 작업은 **코드로 자동화**
- 자신이 첫 번째 사용자 (**도그푸딩**)

### 배드 시그널
- 내가 만든 서비스 대신 다른 걸 쓰고 싶다면? → 즉시 개선!
- 에이전트 결과물을 일일이 수정해야 한다면? → 지침 보강!

---

## 2. 컨텍스트 관리 3원칙

| # | 원칙 | 잘못된 방식 | 올바른 방식 |
|---|------|------------|------------|
| 1 | 스크립트 추출 | AI에게 DB 전체 분석 요청 | 스크립트로 필요한 데이터만 추출 |
| 2 | 배치 분할 | 100개 한꺼번에 처리 | 10개씩 나눠서 처리 |
| 3 | 파일 저장 | 긴 결과를 대화에 출력 | 파일로 저장하고 경로만 공유 |

### 컨텍스트 부족 증상
- AI 응답이 갑자기 짧아짐
- 이전 대화 내용을 잊어버림
- 같은 실수 반복
- 작업 완수 못하고 멈춤

---

## 3. 피드백 루프

### 기존 방식 (비효율)
```
실수 발견 → 별도 문서에 기록 → 나중에 참조 (잊어버림)
```

### 최수민님 방식
```
실수 발견 → 에이전트 .md 열기 → 지침에 직접 추가 → AI가 즉시 학습
```

### 에이전트 파일 구조
```markdown
## 지침
[구체적인 작업 지침 - 피드백 반영하여 계속 추가]

## 자가 검증
[작업 완료 후 체크리스트]

## 피드백 기록 (지침에 반영됨)
| 날짜 | 문제 | 해결 | 지침 추가 내용 |
```

---

## 4. 자가 검증 4단계

### 1단계: 도그푸딩
- 만든 사람이 첫 번째 사용자
- 직접 써보고 불편함 발견

### 2단계: 스킬에 검증 로직 포함
- 스킬 만들 때 "검증도 할 수 있게" 설계
- 에이전트가 작업 후 스스로 체크

### 3단계: 결과 기반 지침 보강
- 오류 발견 → 에이전트 .md에 즉시 반영
- 한 번 수정으로 끝내지 않고 지침에 영구 반영

### 4단계: 파이프라인 최종 종합
- 마지막에 "종합 검증" 단계
- 전체 정합성 확인

---

## 5. 프롬프트 → 에이전트 승격

### 과정
```
1. prompts/ 폴더에 저장
2. {{변수}} 형태로 파라미터화
3. 3회 이상 사용 시 agents/로 승격
4. 지침/자가검증/피드백 섹션 추가
```

### 에이전트 전환 메모
프롬프트 파일에 다음 섹션 추가:
```markdown
## 에이전트 전환 메모
| 항목 | 값 |
|------|-----|
| 현재 사용 횟수 | 0회 |
| 전환 임계값 | 3회/일 |
| 후보 에이전트명 | xxx-agent |
```

## 6. 지식 관리

| 기능 | 에이전트 |
|------|---------|
| 검색 | docs-searcher |
| 분석 | knowledge-analyzer |

### 폴더 구조
```
docs/
├── conversations/  # 대화 기록
├── learnings/      # 학습 내용
├── cheatsheets/    # 치트시트
```
