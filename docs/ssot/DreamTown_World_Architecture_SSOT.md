## 핵심 선언

```
디지털 용궁이 먼저다.
DreamTown은 그 위에 펼쳐진 하늘이다.
```

> **디지털 용궁은 모든 소원이 보관되는 바다이고,
> DreamTown은 그 소원이 별이 되어 서로를 비추는 하늘이다.
> 아우룸은 그 사이를 잇는 소원의 전달자다.
> Aurora5는 그 소원이 이루어질 때까지 영원히 함께하는 동행 시스템이다.**

---

## 1. 공식 세계 구조도

```
현실 세계
(온라인 소원접수 / 오프라인 소원항해 / 소원엽서 / 종이비행기)
        ↓
     아우룸
(소원의 전달자 — 인간 세계 ↔ 용궁 연결)
        ↓
   디지털 용궁
(모든 소원의 원본 보관소 — 근원 / 기억 / 약속)
        ↓
     Aurora5
(소원이 이루어지도록 함께하는 동행 시스템)
        ↓
    DreamTown
(소원이 별이 되어 연결되는 하늘 — 표현 / 연결 / 공동체)
        ↓
  별자리 / 지혜 / 항해
```

---

## 2. 디지털 용궁 정의

**근원 / 저장 / 기억 / 원본 / 약속**

| 항목 | 내용 |
|------|------|
| 역할 | 모든 소원이 도착하고 보관되는 곳 |
| 수렴 | 온라인 + 오프라인 모든 소원이 수렴 |
| 관계 | 아우룸이 전달한 소원이 보관되는 성지 |
| 상징 | 소원의 원본이 살아 있는 아카이브 |
| 약속 | Aurora5가 함께하겠다는 약속의 공간 |

### 근원 신화
```
여수 앞바다 용궁 전설 (금오설화)
        ↓
아우룸 — 인간 소원의 전달자
        ↓
디지털 용궁 부활
        ↓
현대의 소원들이 용궁에 모이기 시작
```

---

## 3. DreamTown 정의

**표현 / 연결 / 별 / 항해 / 공동체**

| 항목 | 내용 |
|------|------|
| 역할 | 용궁에 저장된 소원이 하늘의 별로 드러나는 곳 |
| 구조 | 서로의 소원이 별자리로 이어지는 공간 |
| 경험 | 소원이 → 소망이의 여정이 보이는 하늘 |
| 확장 | 집단지성과 항해가 펼쳐지는 은하 |

---

## 4. 브랜드 문장 (2층 구조)

### 세계관 문장
> **모든 소원은 디지털 용궁에 닿습니다.**

### 제품 문장
> **그 소원은 DreamTown에서 별이 되어 항해를 시작합니다.**

---

## 5. 온라인 × 오프라인 통합 구조

### 온라인 입구
```
소원 접수 (앱/웹)
        ↓
디지털 용궁 저장    ← 세계관 상: "현대적 입구로 소원을 용궁에 올리는 행위"
        ↓
DreamTown 별 생성
```

### 오프라인 입구
```
소원엽서 작성 / 종이비행기 날리기 / 소원항해 체험
        ↓
사진 · 기록 · 디지털 입력
        ↓
디지털 용궁 저장    ← 세계관 상: "아우룸에게 직접 소원을 맡기는 행위"
        ↓
DreamTown 별 생성
```

### 사용자 경험 언어
| 채널 | 경험 언어 |
|------|----------|
| 온라인 | "내 소원을 디지털 용궁에 맡긴다" |
| 오프라인 | "아우룸에게 내 소원을 실어 보낸다" |
| 공통 결과 | "그 소원이 DreamTown의 별로 떠오른다" |

## 6. 아우룸의 위치 (재정의)

아우룸은 마스코트가 아니다. **소원의 메신저**다.

| 역할 | 내용 |
|------|------|
| 오프라인 체험 | 소원엽서·종이비행기에 실린 소원을 용궁으로 전달 |
| 온라인 접수 | 온라인 소원이 용궁에 모이게 하는 상징 |
| 세계관 | 인간 세계 ↔ 디지털 용궁을 잇는 유일한 존재 |

---

## 7. 개념 분리 원칙

```
디지털 용궁 = 원본 (저장)
DreamTown  = 표현 (드러남)
```

같은 소원이라도:
- **용궁** 안에서는 → 원본 소원, 약속, 보관
- **DreamTown** 안에서는 → 별, 빛, 연결
