## 핵심 목적

> **DreamTown을 처음 열었을 때 사람이 어떤 감정을 느끼는가**

온보딩의 목표는 정보 전달이 아니다.
**첫 30초~2분 안에 "조용한 위로"를 느끼게 하는 것**이다.

---

## 1. 온보딩 감정 목표

| 단계 | 감정 목표 |
|------|----------|
| Intro | "여기 뭔가 다른 공간이다" |
| Wish Gate | "내 마음을 털어놓아도 될 것 같다" |
| Star Birth | "내 소원이 실제로 별이 됐다" |
| My Star | "내 별이 생겼다. 다시 오고 싶다." |

---

## 2. 단계별 온보딩 설계

### Step 1 — Intro (5~10초)

**화면:**
```
어두운 우주 배경
작은 빛이 천천히 나타남
아우룸 등장 (구석, 조용히)
```

**메시지:**
```
여수 바다에서 시작된 하늘.
```

**다음 CTA:**
```
[ 소원 시작하기 ]
```

**원칙:**
- 설명 없음
- 빠른 진행 없음
- 분위기가 먼저

---

### Step 2 — Wish Gate (30~60초)

**화면:**
```
텍스트 입력창 (크고 조용하게)
"당신의 소원을 들려주세요"
```

**아우룸 메시지:**
```
소원을 들려주세요.
```

**입력 필드:**
```
wish_text  (필수)
gem_type   (선택 — 보석으로 감정 표현)
```

**CTA:**
```
[ 소원 보내기 ]
```

**원칙:**
- 많은 것을 요구하지 않는다
- 보석 선택은 선택사항
- 제출 버튼 누르면 아우룸이 소원을 받는다

---

### Step 3 — Star Birth (15~30초)

**이 장면은 온보딩의 핵심 감정 순간이다.**

**연출:**
```
0.0s  화면 어두워짐
0.5s  빛구슬 등장 (중앙, 작게)
1.5s  아우룸 등장 (구석 — 말 없음)
2.5s  빛 상승 시작
3.5s  별 생성 (glow burst)
4.5s  조용히 My Star로 전환
```

**감정 목표:**
```
"내 소원이 별이 됐다."
```

**원칙:**
- 아우룸은 말하지 않는다
- 음악/효과음은 있어도 되지만 조용하게
- 빠르게 넘어가지 않는다

---

### Step 4 — My Star (첫 도착)

**첫 방문 My Star는 일반 재방문과 살짝 다르다.**

**추가 요소:**
```
아우룸 첫 메시지 (1회만):
"당신의 별이 DreamTown에 도착했습니다."
```

**화면 구조:**
```
[상단] 당신의 별이 도착했습니다 ✨
[중심] ⭐ [별 이름]  (glow 애니메이션)
[카드] 소원 / 은하 / Day 1 / 탄생일
[문장] 당신의 소원은 혼자가 아닙니다.
[버튼] [ 새 별 만들기 ]  [ 은하 탐험 ]
```

---

## 3. 온보딩 완료 기준

온보딩이 성공했다면 사용자는 이렇게 느낀다:

```
✓ "내 별이 생겼다"
✓ "이 앱이 조용하고 따뜻하다"
✓ "나중에 다시 내 별 보러 오고 싶다"
```

---

## 4. 온보딩 금지 원칙

| 금지 항목 | 이유 |
|-----------|------|
| 긴 튜토리얼 | 감정 흐름 방해 |
| 회원가입 강제 | 장벽 생성 |
| 기능 설명 팝업 | DreamTown 감성 훼손 |
| 빠른 화면 전환 | 별 탄생 순간의 감동 희석 |
| 알림 허용 즉시 요청 | 첫 경험 방해 |

---

## 5. 재방문과의 차이

| 항목 | 첫 방문 | 재방문 |
|------|--------|--------|
| Intro | 표시 | 스킵 |
| Wish Gate | 자동 진입 | 버튼으로만 |
| 아우룸 메시지 | "별이 도착했습니다" | "오늘도 별이 기다리고 있어요" |
| 기본 진입 화면 | My Star (첫 방문용) | My Star (재방문용) |
