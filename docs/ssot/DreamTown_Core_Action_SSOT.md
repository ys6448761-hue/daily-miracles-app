## 핵심 선언

> **DreamTown에서 가장 중요한 행동은 "내 별 보기 (View My Star)"이다.**

DreamTown은 별을 만드는 서비스가 아니다.
**이미 만들어진 자신의 별을 다시 보며 마음을 정리하는 서비스**이다.

---

## 1. Core Action 정의

```
Core Action = View My Star (내 별 보기)
```

### 의미
내 별은 다음을 상징한다:

| 상징 | 의미 |
|------|------|
| 나의 소원 | 내가 원하는 것 |
| 나의 마음 | 지금 내 상태 |
| 나의 작은 희망 | 포기하지 않은 것 |

따라서 **"내 별 보기"**는 단순한 화면 이동이 아니다.

```
내 마음을 다시 바라보는 행동
```

---

## 2. 핵심 화면 우선순위

DreamTown에서 사용자가 가장 많이 보게 될 화면:

```
1순위 — My Star (내 별 공간)   ← Core Action 화면
2순위 — Dream Plaza (소원꿈터 광장)
3순위 — Wish Gate (소원 게이트)
4순위 — Galaxy Map (은하 지도)
```

> My Star 화면이 DreamTown의 **핵심 화면**이다.
> Galaxy도, Wish Gate도 아니다.

---

## 3. 사용자 행동 구조 (4단계)

DreamTown의 사용자 행동은 4단계로 구성된다:

```
1. 소원 남기기 (Wish)        ← 시작
       ↓
2. 내 별 보기 (View My Star) ← Core Action (가장 중요)
       ↓
3. 하늘 탐험하기 (Explore)   ← 연결·공명
       ↓
4. 나눔하기 (Share)          ← 소망이로 성장
```

이 4개 행동이 잡히면 서비스 방향은 흔들리지 않는다.

---

## 4. My Star 화면 — 핵심 요소

My Star 화면에서 사용자가 볼 수 있어야 하는 것:

| 요소 | 내용 |
|------|------|
| 내 별 비주얼 | 소원그림 기반 별 |
| 소원 텍스트 | 처음 남긴 소원 원문 |
| 별 탄생일 | 별이 된 날짜 |
| 별 에너지 | 기적 기록 누적 |
| 여정 현황 | 항해 진행 상태 |
| 아우룸 메시지 | 별을 바라보며 건네는 말 |

---

## 5. My Star 감성 원칙

My Star 화면을 설계할 때 지켜야 할 감성 원칙:

```
조용하다   — 빠른 피드가 아니다
따뜻하다   — 내 별만 있는 공간
사적이다   — 나만 아는 마음의 공간
살아있다   — 별 에너지가 천천히 자란다
```

**금지 패턴:**
- 빠른 인터랙션 (탭/슬라이드 연속)
- 숫자 게임화 (포인트 강조)
- SNS 피드 느낌 (리스트 스크롤)

---

## 6. 반복 방문 설계 원칙

My Star는 사용자가 **반복해서 돌아오는** 화면이어야 한다.

반복 방문을 유도하는 3가지 요소:

| 요소 | 설명 |
|------|------|
| **별 에너지 성장** | 기적 기록할수록 별이 밝아진다 |
| **아우룸 메시지 변화** | 방문할 때마다 다른 말을 건넨다 |
| **여정 진행 감각** | 항해가 진행될수록 별이 달라진다 |
