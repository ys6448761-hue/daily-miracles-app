# SSOT-LOOP-001_Seal_Return_Unlock_Loop_Guide

## 목적

본 문서는 DreamTown과 하루하루의 기적의 핵심 사업 루프를 정의한다.

핵심은 콘텐츠 소비가 아니라 감정의 재방문이다.

사용자는 소원을 남기고, 감정을 봉인하고, 시간이 지난 뒤 같은 장소 또는 연결된 장소에서 다시 해제한다.

---

## 핵심 루프

```
소원 입력
   ↓
감정 봉인
   ↓
시간잠금
   ↓
기다림
   ↓
장소 재방문
   ↓
해제
   ↓
회복 확인
   ↓
공유 또는 다음 소원
```

---

## 핵심 정의

### 봉인

사용자의 감정과 소원을 별, 이미지, 영상, 장소에 저장하는 행위.

### 시간잠금

즉시 소비되지 않도록 일정 기간 열리지 않게 하는 장치.

### 재방문

사용자가 디지털 또는 실제 장소로 다시 돌아오는 행위.

### 해제

봉인했던 감정과 소원을 다시 열어 확인하는 순간.

### 회복

사용자가 "조금 괜찮아졌다"라고 느끼는 상태.

---

## DreamTown 적용

DreamTown은 장소 기반 감정 회복 항로다.

장소는 배경이 아니라 감정 저장소다.

| 장소 | 역할 |
|------|------|
| 여수엑스포역 | 도착 |
| 엑스포 바닷길 | 호흡 |
| 이순신광장 | 연결 |
| 케이블카 | 상승 |
| 종포해양공원 | 쉼 |
| 하멜등대 | 소원 |
| 호텔 | 안식 |

---

## 하루하루의 기적 적용

하루하루의 기적은 소원 기반 감정 성장 항로다.

소원은 단순 입력값이 아니라 시간 속에서 자라는 별이다.

사용자는 소원을 봉인하고, 7일 / 30일 / 90일 단위로 다시 확인한다.

---

## 검증 지표

반드시 측정해야 할 지표는 다음과 같다.

- 봉인 완료율
- 7일 후 재접속률
- 30일 후 재접속률
- 90일 후 재접속률
- 현장 재방문 의향
- 실제 현장 해제율
- 해제 후 공유율
- 해제 후 결제율
- 다음 소원 생성률

---

## 핵심 판단 기준

이 루프가 돌면 사업이다.

이 루프가 돌지 않으면 예쁜 콘텐츠다.

---

## 금지 사항

- 즉시 소비형 콘텐츠 중심 운영 금지
- 무한 이미지 생성 중심 운영 금지
- 단체숏츠를 루프 검증 전 핵심 KPI로 삼는 것 금지
- 관광 홍보 서비스로 변질 금지
- 장소를 단순 배경으로 취급 금지
- 감정을 결제 유도 장치로만 사용하는 것 금지

---

## 우선순위

1. 봉인 루프 검증
2. 재방문율 측정
3. 해제 경험 개선
4. 공유 루프 설계
5. 단체숏츠 확장
6. 중국 단체 확장
7. 오프라인 제휴 확장

---

## 최종 원칙

DreamTown과 하루하루의 기적의 핵심은 AI 콘텐츠가 아니다.

핵심은

**감정이 시간과 장소를 지나 다시 열리는 경험이다.**

```
봉인 → 시간 → 장소 → 재방문 → 해제
```

이 루프가 모든 제품 판단의 기준이다.
