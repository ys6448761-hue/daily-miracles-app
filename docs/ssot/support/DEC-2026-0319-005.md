## 결정

포스트카드 공유 화면은 결과 설명 화면이 아니라
**감정 장면 화면**으로 구현한다.

---

## 고정 사항

### 핵심 철학
> "오늘 이런 걸 했어요"가 아니라
> "오늘 이런 순간이 있었어요"를 전달한다.

### 절대 금지
- ❌ 통계/점수/streak 노출
- ❌ "도전/성장/관계/치유" 라벨 직접 표시
- ❌ 결과 요약형 텍스트 (예: "오늘 3가지를 완료했습니다")
- ❌ 앱 기능 설명 문구

### 카드 구성 규칙
- 메인 문장: `message` 1개만 — 감정 장면의 중심
- 보조 문장: `growthLine` 1개 (선택적, 작게)
- 브랜드: 하단에 `DreamTown / 하루하루의 기적` 만 표기 (최소)

### 카드에서 절대 노출하지 않는 것
- direction (north/east/west/south)
- 단계(stage) / 로그 구조 / 시스템 정보
- 기능 설명 / 결과 요약 문구

### 컴포넌트 구조 (예정)
```
pages/ShareCard.jsx                    ← 공유 화면 진입점
  └─ components/PostCard.jsx           ← 카드 UI (캡처 대상)
  └─ components/PostCardBackground.jsx ← 수채화 배경
```

---

## 루미 최종 의견

> 공유는 "내가 뭘 했는지"를 알리는 기능이 아니라,
> "오늘 내 별에 어떤 빛이 남았는지"를 건네는 기능이어야 한다.

## 이유

- 공유는 "앱 홍보"가 아니라 "감정 전달"이어야 한다
- 받는 사람이 "무슨 앱인지"보다 "어떤 순간인지"를 먼저 느껴야 한다
- 장면 중심 카드는 자연스러운 바이럴로 이어진다
- 시스템 정보 노출 → 경험이 깨진다
