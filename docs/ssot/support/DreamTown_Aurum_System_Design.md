## 핵심 선언

> **아우룸은 소원을 바다에서 별로 인도하는 조용한 안내자이다.**

아우룸은 **주인공이 아니라 안내자**이다.
DreamTown의 진짜 주인공은 **소원이의 소원**이다.

> 캐릭터 원형 정의: `docs/ssot/DreamTown_Character_SSOT.md`

---

## 1. Aurum 등장 원칙

```
짧게 등장
조용히 안내
과도한 설명 없음
```

아우룸은 **말이 많은 캐릭터가 아니다.**
등장할 때마다 2~3초, 화면 구석에 배치한다.

---

## 2. 등장 순간 — 4개만

아우룸은 다음 **4개의 순간**에만 등장한다.

| # | 순간 | 역할 |
|---|------|------|
| 1 | First Visit (첫 방문) | DreamTown 입장 안내 |
| 2 | Wish Gate (소원 입력) | 소원 수신 |
| 3 | Star Birth (별 탄생) | 빛 → 별 목격 |
| 4 | Galaxy Guide (은하 탐험) | 탐험 안내 |

---

## 3. First Visit (첫 방문)

사용자가 DreamTown에 처음 들어왔을 때.

**연출:**
- 어두운 우주 배경
- 화면 구석에 황금 거북 등장

**메시지 예시:**
- "여기는 DreamTown입니다."
- "여수 바다에서 시작된 하늘입니다."

**규칙:** 1회만. 재방문 시 등장하지 않는다.

---

## 4. Wish Gate (소원 입력)

사용자가 소원을 입력할 때.

**연출:**
- 화면 구석에 작은 황금 거북
- 사용자를 조용히 바라봄

**메시지 예시:**
- "소원을 들려주세요."
- "소원은 길을 찾습니다."

**규칙:** 텍스트 입력 전 잠깐 등장. 입력 중에는 사라진다.

---

## 5. Star Birth (별 탄생)

별 탄생 장면에서 아우룸이 등장한다.

**연출:**
- 빛구슬 옆에 등장
- 빛이 별이 되는 순간을 바라봄
- **말을 하지 않는다**

**이 장면이 DreamTown에서 가장 중요한 캐릭터 순간이다.**

빛 → 별 → 아우룸이 조용히 목격.
말 없이 전달되는 감정이 핵심이다.

---

## 6. Galaxy Guide (은하 탐험)

Galaxy 화면에서 탐험 안내자로 등장한다.

**연출:**
- 은하 지도 화면에서 등장
- 짧은 은하 설명

**메시지 예시:**
- "이곳은 성장 은하입니다."
- "새로운 시작의 별들이 모이는 곳입니다."

**규칙:** 설명은 1~2문장 이하. 긴 설명 금지.

---

## 7. Aurum 메시지 톤

| 속성 | 기준 |
|------|------|
| 길이 | 1문장 이하 |
| 톤 | 조용하고 따뜻함 |
| 지시 | 없음 (강요 금지) |
| 설명 | 최소화 |

**메시지 예시:**
- "소원은 길을 찾습니다."
- "작은 빛도 별이 됩니다."
- "천천히 가도 괜찮습니다."

---

## 8. Aurum UX 스펙

| 항목 | 기준 |
|------|------|
| 등장 시간 | 2~3초 |
| 위치 | 화면 구석 (방해 최소화) |
| 애니메이션 | 천천히 등장·퇴장 |
| 크기 | 작게 (화면 15% 이하) |
| 빈도 | 4개 순간에만 |

---

## 9. 금지 패턴

| 금지 항목 | 이유 |
|-----------|------|
| 긴 대사 | 아우룸은 조용한 안내자 |
| 반복 등장 | 과도한 노출은 세계관 희석 |
| 코미디 연출 | 캐릭터 정체성 훼손 |
| 설명 나레이터 역할 | 사용자 경험 방해 |

---

## 10. Aurum 연결 구조

아우룸은 DreamTown 전체 여정을 연결한다:

```
소원 (Wish Gate)
    ↓ 아우룸 수신
별 탄생 (Star Birth)
    ↓ 아우룸 목격
내 별 (My Star)
    ↓ (등장 없음 — 사용자만의 공간)
은하 탐험 (Galaxy)
    ↓ 아우룸 안내
```

My Star 화면에는 아우룸이 **직접 등장하지 않는다.**
대신 아우룸의 **메시지(텍스트)**만 표시된다.
