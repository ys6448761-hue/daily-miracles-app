## 결정

DreamTown Galaxy 선택 화면의 구현 아키텍처를
`features/galaxy` 3층 구조로 고정한다.

---

## 고정 사항

### 구조
- 구현 기준: `features/galaxy/` (constants / hooks / components)
- 페이지 레벨: 얇은 래퍼만 허용 (라우팅 + 탭바)

### 불변 규칙
1. "도전/성장/관계/치유" 텍스트를 UI에 노출하지 않는다
2. 별의 감각 차이는 `useStarMotion.js` 외부에서 만들지 않는다
3. 선택 전환 로직은 `useGalaxySelection.js` 상태머신을 기준으로 유지한다
4. 별 배치 상수, 모션 프리셋, 선택 상태머신, 전환 컴포넌트는 분리 유지한다

---

## 이유

- 나중에 Flutter로 이식해도 구조가 무너지지 않는다
- 감성보다 구조가 먼저 서야 UX 수정이 안전하다
- SelectionTransition은 가장 많이 수정되는 구간이므로 반드시 독립 파일 유지
