## 결정

Galaxy 선택 후 이어지는 로그 시스템을
**direction 기반 반응형 기록 UI** 3단계 구조로 고정한다.

---

## 고정 사항

### 구조
```
선택 (Galaxy)
  ↓
1단계: feeling     — 오늘 어떤 느낌이었나요?
  ↓
2단계: helpTag     — 무엇이 도움이 됐나요?
  ↓
3단계: growthLine  — 오늘을 한 줄로 남긴다면?
```

### 불변 규칙
1. 자유 입력형 일기장이 아니다 — 모든 선택지는 문장 풀에서만 노출
2. 모든 문장은 `galaxyLogCopy.js`의 direction별 풀에서만 가져온다
3. direction 라벨(도전/성장/관계/치유)을 사용자에게 직접 노출하지 않는다
4. 한 화면 한 선택 — 단계를 한 화면에 몰아넣지 않는다
5. 문장 SSOT 규칙(DEC-2026-0319-002 §Copy 규칙) 동일 적용

### 데이터 구조
```js
{
  direction,    // 시스템용
  feeling,      // 사용자 선택값
  helpTag,      // 사용자 선택값
  growthLine,   // 사용자 선택값
  timestamp,
  sessionId,
}
```

---

## 이유

- 자유 입력 → 부담감, 공백 발생
- 선택형 → 진입 장벽 낮음 + 데이터 구조화 가능
- direction별 문장 풀 → 맥락 일관성 유지
- 3단계 분리 → 한 화면에 과부하 없음
