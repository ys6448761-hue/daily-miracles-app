## 핵심 목적

> **사용자가 자신의 소원을 다시 바라보며 마음에 작은 위로를 느끼는 것**

My Star 화면은 DreamTown에서 **가장 자주 열리는 화면**이다.

---

## 진입 구조 (CEO 확정 2026-03-12)

```
첫 방문     Intro → Wish Gate → Star Birth → My Star
재방문      앱 열기 → My Star (기본 진입 화면)
```

- 재방문 시 Home을 거치지 않는다
- Today Message는 My Star 화면 안에 배치
- 주요 행동 버튼(새 소원 / 은하 탐험)도 My Star 화면 안에 배치

---

## 1. 화면 감정 방향

My Star 화면이 전달해야 하는 감정:

| 감정 | 설명 |
|------|------|
| 조용함 | 빠른 피드가 아닌 숨을 고르는 공간 |
| 따뜻함 | 나만을 위한 별이 있다는 느낌 |
| 안정감 | 소원이 안전하게 보관되어 있다는 확신 |
| 작은 희망 | 별이 살아있고 자라고 있다는 감각 |

```
잠깐 숨을 고르는 공간
```

---

## 2. 화면 구조

```
─────────────────────────
  상단 인사 메시지
─────────────────────────
  내 별 (중심 요소)
  ⭐ + 별 이름
─────────────────────────
  별 정보 카드
  소원 / 은하 / 성장 단계 / 탄생일
─────────────────────────
  위로 메시지 (짧은 문장)
─────────────────────────
  행동 버튼 (최대 2개)
─────────────────────────
```

---

## 3. 상단 영역 — 인사 메시지

목적: 사용자가 DreamTown에 **부드럽게 들어오게 하는 것**

예시 문장:
- "오늘도 별이 기다리고 있어요 ✨"
- "당신의 별이 조용히 빛나고 있습니다."

규칙:
- 1줄 이내
- 부드럽고 따뜻한 어조
- 정보 전달 목적이 아님

---

## 4. 별 성장 단계 (Star Growth Stage)

| 단계 | 이름 | 조건 |
|------|------|------|
| 1 | 씨앗 ⭐ | 별 생성 직후 (Day 1~6) |
| 2 | 새싹 ⭐✨ | Day 7 이후 |
| 3 | 빛나는 별 ⭐💫 | Day 30 이후 |
| 4 | 성운 ⭐🌌 | Day 100 이후 |

별은 사용자의 기록에 따라 성장한다.
Prototype에서는 **씨앗 단계**만 구현한다.

---

## 5. 중심 요소 — 내 별

화면에서 **가장 강조되는 요소**이다.

### 표시 요소
| 요소 | 예시 |
|------|------|
| 별 아이콘 | ⭐ (소원그림 기반) |
| 별 이름 | First Wish Star |

### 레이아웃
- 화면 중앙 배치
- 다른 요소보다 크게
- 부드러운 반짝임 애니메이션 적용

### 애니메이션 원칙
| 항목 | 기준 |
|------|------|
| 종류 | 부드러운 반짝임 (glow pulse) |
| 속도 | 빠른 움직임 없음 |
| 반복 주기 | 3~5초 |
| 목적 | 화면을 살아있는 느낌으로 만드는 것 |

---

## 5. 별 정보 카드

별 아이콘 아래 카드 형태로 표시한다.

### 표시 항목
```
소원
[wish_text]

은하
[galaxy_name]

성장 단계
Day [N]

탄생일
[YYYY-MM-DD]
```

### 예시
```
소원
조금 더 나아지기를

은하
Growth Galaxy

성장 단계
Day 1

탄생일
2026-03-12
```

### 규칙
- 카드 UI (모서리 둥글게)
- 정보는 4개 이하
- 숫자 게임화 금지 (포인트 강조 금지)

---

## 6. 위로 메시지

My Star 화면에는 항상 **짧은 위로 메시지**가 표시된다.

DreamTown의 감정 중심 요소이다.

### 예시 문장
- "작은 소원도 별이 됩니다."
- "오늘도 괜찮습니다."
- "당신의 소원은 혼자가 아닙니다."

### 규칙
- 1~2줄 이내
- 방문할 때마다 다른 문장 (아우룸 메시지)
- 조언이나 지시 금지
- 위로·공감 어조만 허용

---

## 7. 행동 버튼

**최대 2개**를 초과하지 않는다.

### 버튼 1 — 새 별 만들기
```
[ 새 소원 남기기 ]
```

### 버튼 2 — 은하 탐험
```
[ 은하 탐험하기 ]
```

### 규칙
- 버튼 3개 이상 금지
- Primary / Secondary 구분
- 버튼 텍스트는 행동 중심 (명사형 금지)

---

## 8. UX 원칙

### 정보보다 감정
정보가 많아지지 않도록 한다.
화면의 목적은 데이터 표시가 아니라 감정 전달이다.

### 조용한 디자인
과한 그래픽을 사용하지 않는다.
별 하나가 화면을 채운다.

### 짧은 체류 허용
사용자는 오래 머무르지 않아도 된다.
30초~1분 체류만으로 충분한 경험을 제공한다.

### 금지 패턴
| 금지 항목 | 이유 |
|-----------|------|
| 빠른 인터랙션 (탭/슬라이드 연속) | 감정 흐름 방해 |
| 숫자 게임화 (포인트 강조) | 소원의 의미 희석 |
| SNS 피드 느낌 (리스트 스크롤) | DreamTown 정체성 훼손 |
| 버튼 3개 이상 | 선택 과부하 |

---

## 9. 반복 방문 설계

My Star는 사용자가 **반복해서 돌아오는** 화면이어야 한다.

| 요소 | 설명 |
|------|------|
| 별 에너지 성장 | 기적 기록할수록 별이 밝아진다 |
| 아우룸 메시지 변화 | 방문할 때마다 다른 말을 건넨다 |
| 여정 진행 감각 | Day N이 올라갈수록 별이 달라진다 |

---

## 10. My Star 화면 최종 정의

```
사용자가 자신의 소원을 다시 바라보며 마음을 정리하는 공간
```

DreamTown 핵심 경험 2장면:

```
1장면: 소원 → 별 탄생 (Star Birth)
2장면: 별 → 다시 보기 (My Star)   ← 이 화면
```

My Star는 사용자가 **돌아오는 집**이다.
