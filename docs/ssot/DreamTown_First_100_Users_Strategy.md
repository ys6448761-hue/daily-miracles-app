## 핵심 원칙

DreamTown 같은 서비스는 작은 커뮤니티에서 성장한다.

```
1명 → 10명 → 100명 → 1,000명 → 10,000명
```

첫 100명이 DreamTown의 **초기 문화를 만든다.**

---

## 1. 첫 100명의 의미

첫 100명은 단순한 사용자가 아니다.

| 역할 | 의미 |
|------|------|
| 감정 검증 | "위로가 됐는가?" 실제 확인 |
| 세계관 공명 | DreamTown 세계관에 공감하는 사람 |
| 첫 Star Share | 자연 바이럴의 씨앗 |
| 피드백 원천 | 제품 개선 방향 결정 |

---

## 2. 타깃 프로파일

첫 100명은 다음 특성을 가진 사람이어야 한다:

```
조금 흔들리지만 포기하지 않은 사람
밤에 혼자 마음을 정리하는 시간이 있는 사람
거창한 조언보다 조용한 공감을 원하는 사람
감성적인 디지털 경험에 열려 있는 사람
```

**20~35세 / 여성 중심** (초기 타깃)

---

## 3. 확보 경로 3가지

### 경로 1 — 직접 관계망 (1~30명)

| 항목 | 내용 |
|------|------|
| 방법 | CEO 개인 네트워크에서 직접 초대 |
| 대상 | 타깃 프로파일에 맞는 지인 |
| 메시지 | "내가 만드는 서비스인데 한번 써봐줄 수 있어?" |
| 목표 | 20~30명 확보 |

---

### 경로 2 — 감성 커뮤니티 (30~70명)

| 항목 | 내용 |
|------|------|
| 채널 | 감성 인스타그램 / 소규모 오픈채팅 / 독서모임 등 |
| 방법 | Star Card 이미지 + "내 소원을 별로 만들었어요" 자연 공유 |
| 목표 | 30~40명 추가 확보 |

---

### 경로 3 — Star Share 바이럴 (70~100명)

| 항목 | 내용 |
|------|------|
| 방법 | 첫 사용자들이 Star Card를 자연스럽게 공유 |
| 트리거 | "이거 뭐야?" → Wish Gate 진입 |
| 목표 | 자연 유입으로 30명 추가 |

---

## 4. 첫 100명 온보딩 원칙

첫 100명에게는 특별한 경험을 제공한다:

```
Founding Star (창립 별)
```

- 첫 100명의 별은 **Founding Star** 뱃지를 받는다
- "DreamTown이 시작될 때 함께한 별"이라는 의미
- 이후 Galaxy Map에서 특별 표시

---

## 5. 첫 100명 성공 기준

| 지표 | 목표 |
|------|------|
| 별 생성 | 100개 이상 |
| My Star 재방문 | 40% 이상 |
| Star Share 자연 발생 | 10건 이상 |
| 위로 경험 응답 | "위로가 됐다" 70% 이상 |

---

## 6. 피드백 수집

첫 100명에게 물어볼 핵심 질문 3가지:

```
1. "이 서비스가 조용한 위로가 됐나요?"
2. "다시 내 별을 보러 오고 싶었나요?"
3. "이 서비스를 한 문장으로 설명한다면?"
```

이 3가지 답변이 DreamTown MVP의 **방향 결정 기준**이 된다.

---

## 7. 타임라인

```
Prototype 완성     → Closed Alpha 5~20명
        ↓
피드백 반영        → Quiet Beta 20~50명
        ↓
Star Share 추가    → Soft Launch 50~100명
        ↓
첫 100명 달성      → Public Launch 준비
```
