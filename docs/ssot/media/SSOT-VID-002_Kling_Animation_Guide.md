# SSOT-VID-002 — DreamTown 영상 생성 가이드

## 목적

Kling, Sora 등 영상 생성 도구가 변경되어도
DreamTown의 감정 표현 품질을 동일하게 유지한다.

> 도구는 교체 가능하다. 감정 기준은 교체 불가능하다.

---

## 1. 영상 생성 원칙

| 항목 | 기준 |
|------|------|
| 장면 길이 | 한 장면 8초 |
| 움직임 속도 | 느리게 |
| 최우선 원칙 | 캐릭터 일관성 |

---

## 2. 금지 항목

다음은 DreamTown 영상에서 절대 사용하지 않는다.

- 과한 액션
- 뮤직비디오 스타일 연출
- 틱톡 스타일 리액션
- 빠른 컷 전환
- 과한 카메라 흔들림 (핸드헬드 과도 사용)

---

## 3. 권장 움직임 목록

소원이의 움직임은 감정을 직접 표현하지 않는다.
장면 자체가 감정을 전달한다.

| 움직임 | 감정 효과 |
|--------|----------|
| 바람 느끼기 | 도착, 열림 |
| 수평선 바라보기 | 소원, 그리움 |
| 천천히 걷기 | 회복, 진행 |
| 깊은 호흡 | 정리, 수용 |
| 작은 미소 | 희망 시작 |
| 빛 바라보기 | 별, 연결 |

---

## 4. 에피소드별 영상 기준

### EP01 Original

**주요 행동:** 한숨 4회

**관객 반응:**
- 눈물 난다
- 슬퍼 보인다

**감정 수치:** 슬픔 60 / 희망 40

---

### EP01 Positive 수정 방향

Original 대비 아래 방향으로 조정한다.

| 항목 | Original | Positive |
|------|----------|----------|
| 한숨 | 4회 | 감소 |
| 미소 | 없음 | 작은 미소 추가 |
| 바다 연결 | 약함 | 강화 |
| 감정 | 슬픔 우세 | 희망 우세 |

> Positive 버전은 Original을 **부정하지 않는다.**
> 같은 장소, 같은 소원이가 조금 더 앞으로 나아간 버전이다.

---

## 5. 프롬프트 작성 원칙

영상 생성 프롬프트 작성 시 아래 구조를 따른다.

```
[장소] + [소원이 상태] + [움직임] + [감정 톤] + [스타일 잠금]
```

**예시:**
```
여수 하멜등대 앞, 긴 검은 머리의 소원이가 천천히 수평선을 바라본다.
바람이 머리카락을 흔든다. 카메라는 뒤에서 천천히 당긴다.
수채화 스타일, 지브리 감성, 슬픔 60 희망 40.
```

## 6. 도구별 적용 메모

| 도구 | 강점 | 주의 |
|------|------|------|
| Kling | 움직임 자연스러움 | 과한 드라마틱 방지 설정 필요 |
| Sora | 공간감 | 캐릭터 일관성 별도 확인 필요 |

---

## 7. 텍스트 및 클로즈업 규칙

AI 영상 생성 시 텍스트가 포함된 소품은 클로즈업하지 않는다.

| 항목 | 규칙 |
|------|------|
| 가방·간판·포스터·책 표지 등 | 중경 또는 원경에서만 사용 |
| 문구·글자 | 후반 편집에서 삽입 (AI 생성 금지) |

### 엔딩 장면 필수 규칙

| 항목 | 기준 |
|------|------|
| 텍스트 소품 클로즈업 | 금지 |
| 주인공 완전 소멸 | 금지 |
| 등대 불빛 | 유지 |
| 소원이 | 엔딩 장면에 유지 |
| 감정 | 희망 여운 유지 |
