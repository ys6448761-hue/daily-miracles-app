# HUMAN_FIRST
## 인간이 먼저다

**레이어**: Constitution (Layer 1 — 절대 불변)  
**작성**: 2026-05-16  
**상태**: LOCKED — 담당자 교체 시에도 유지. CEO만 변경 가능.

---

```yaml
why_this_exists:
  DreamTown은 콘텐츠 소비 플랫폼이 아니다.
  소원이가 자신의 소원을 다시 만나는 곳이다.
  알고리즘이 인간을 이길 수 없다.
  인간의 소원이 항상 먼저다.
```

---

## 원칙

DreamTown의 모든 기능, 모든 문서, 모든 코드는  
**"이게 소원이에게 도움이 되는가?"** 라는 질문을 먼저 통과해야 한다.

기능이 아무리 기술적으로 훌륭해도,  
소원이가 자신의 소원에서 멀어지게 만들면 그 기능은 틀렸다.

---

## 구체적 적용

| 상황 | HUMAN_FIRST 적용 |
|------|----------------|
| 신규 기능 기획 | "이게 소원이의 소원 여정을 돕는가?" |
| UI 설계 | 압박 0 — 긴급/제한/카운트다운 없음 |
| 메시지 문구 | 단정 어투 없음, 강요 없음 |
| 안전 시스템 | RED → 즉시 관리자 연결. 상업 로직 우선 금지 |
| 결과물 파생 | 감정 착취 목적의 파생 금지 |

---

## 이 원칙이 지켜지지 않을 때

다음이 발생하면 HUMAN_FIRST 위반 신호다:

- "참여율을 높이려고" → 소원이를 불안하게 만드는 기능
- "전환율을 높이려고" → 소원 입력에 시간 제한
- "리텐션을 높이려고" → 감정적 의존성 유발 메시지
- "콘텐츠를 늘리려고" → 소원이의 허락 없이 소원 내용 활용

이 네 가지가 나오면 멈추고 CEO에게 보고한다.
