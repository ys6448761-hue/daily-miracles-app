---

```yaml
why_this_exists:
  DreamTown은 소원이의 감정 취약성을 이용하지 않는다.
  소원이가 위로를 얻기 위해 DreamTown에 의존하게 만들지 않는다.
  우리가 만드는 것은 감정 착취가 아니라 자기 소원과의 재연결이다.
```

---

## 원칙

DreamTown은 소원이가 강해지도록 돕는다.  
DreamTown 없이는 살 수 없게 만들지 않는다.

---

## 금지 패턴 (엄격 금지)

| 금지 행위 | 이유 |
|----------|------|
| 긴급 메시지 ("오늘 밤 12시까지만!") | 압박 = 감정 착취 |
| 제한 수량 강조 | 인위적 불안 유발 |
| "당신만을 위한" 과잉 개인화 | 의존성 유발 언어 |
| 오랜 이탈 후 죄책감 유발 푸시 | 감정 조작 |
| RED 상황에서 상업 메시지 | 취약자 착취 |
| VIP 패턴 감지 후 프리미엄 업셀 | 간절함 상업화 |

---

## 안전 시스템과의 연결

이 원칙은 코드로도 구현되어 있다:

```
services/safetyFilter.js     ← RED/YELLOW/GREEN 분류
services/silentResponseService.js  ← YELLOW/VIP = 조용한 응대, 상업 로직 차단
config/silent-response-messages.json  ← 위로 메시지 (강요 없음)
```

RED 상황 → 상업 로직 차단 → 관리자 연결  
YELLOW/VIP → 3.5초 침묵 → 소원이가 먼저 다음 행동 결정

---

## 파생 결과물 기준

스토리북, 기적영상, 쇼츠가 다음 목적으로 만들어져서는 안 된다:

- 이탈 방지 (소원이를 붙잡기 위해)
- 더 많은 소원 입력 유도 (소원이를 쓰게 만들기 위해)
- 감정 폭발 유발 (공유·바이럴 목적)

올바른 목적:

- 소원이가 자기 소원을 다시 기억하도록
- 소원이가 현실에서 한 발짝 내딛도록
- 소원이가 혼자가 아님을 느끼도록

마케팅 압박, 성장 지표, 리텐션 목표가 이 원칙과 충돌하면  
항상 이 원칙이 이긴다.

지표가 이 원칙을 이기도록 만들면 DreamTown이 아니다.
