## 핵심 선언

> **DreamTown에서 가장 중요한 순간은 사용자가 자신의 별을 다시 바라보는 순간이다.**

DreamTown의 핵심 경험은 "별을 만드는 것"이 아니라
**이미 만들어진 별을 다시 보며 마음을 정리하는 것**이다.

---

## 1. Key Moment 감정 구조

사용자가 My Star 화면을 열 때 일어나는 감정:

```
내 소원이 아직 여기 있다
내 마음이 사라지지 않았다
나는 혼자가 아니다
```

이 순간 DreamTown은 단순한 앱이 아니라

> **마음을 잠깐 내려놓는 공간**

이 된다.

---

## 2. DreamTown 경험 구조

```
소원 입력
    ↓
별 탄생 (Star Birth)     ← 첫 감정 순간
    ↓
내 별 (My Star)
    ↓
다시 보기 (Key Moment)   ← 가장 중요한 순간
```

이 중 **가장 중요한 순간은 "다시 보기"**이다.

---

## 3. Star Birth와 Key Moment 관계

Star Birth는 DreamTown의 **첫 감정 순간**이다.

```
소원 → 빛 → 별 탄생
```

하지만 DreamTown의 핵심은 이 다음이다.

```
시간이 흐름
    ↓
사용자가 다시 앱을 엶
    ↓
내 별을 다시 봄   ← Key Moment
```

이 순간 DreamTown은 **위로 서비스**가 된다.

---

## 4. Key Moment 화면

Key Moment는 **My Star 화면**에서 발생한다.

| My Star 화면 역할 | 설명 |
|------------------|------|
| 내 별 강조 | 화면 중심에 별 배치 |
| 소원 다시 보기 | 처음 남긴 소원 원문 표시 |
| 짧은 위로 메시지 | 아우룸의 감정 메시지 |

> 상세 UX: `docs/design/DreamTown_My_Star_UX_Master_Design.md`

## 5. DreamTown 감정 흐름

```
소원
    ↓
희망
    ↓
별 탄생
    ↓
시간 경과
    ↓
다시 보기
    ↓
위로      ← DreamTown이 제공하는 핵심 가치
```

---

## 6. 제품 설계 우선순위

Key Moment 정의에 따른 화면 역할:

| 화면 | 역할 | 우선순위 |
|------|------|---------|
| **My Star** | 핵심 화면 — Key Moment 발생 장소 | 1순위 |
| **Home** | 내 별로 돌아오는 입구 | 2순위 |
| **Wish Gate** | 행동 시작 | 3순위 |
| **Galaxy** | 확장 경험 | 4순위 |

---

## 7. UX 설계 기준 질문

My Star 화면을 포함한 모든 화면 설계 시 이 질문을 기준으로 한다:

```
이 화면이 사용자가 다시 돌아오고 싶은 공간인가?
```

---

## 8. DreamTown 핵심 4요소

```
Core Emotion    → 위로
Core Experience → 별 생성
Core Action     → 내 별 보기
Key Moment      → 내 별 다시 보기
```

이 4개가 맞물리면 서비스 방향은 흔들리지 않는다.

---

## 9. DreamTown 핵심 문장

```
DreamTown은 소원을 별로 보내고, 다시 그 별을 보러 오는 서비스다.
```

이 문장은 **제품 / 세계관 / UX**를 동시에 설명한다.
