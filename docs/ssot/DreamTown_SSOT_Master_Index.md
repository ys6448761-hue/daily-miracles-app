## SSOT 작성 원칙

1. 하나의 문서는 하나의 기준만 가진다
2. 같은 내용의 SSOT를 여러 개 만들지 않는다
3. 변경 시 Version을 올린다
4. 상태(Draft / Review / Confirmed)를 표시한다

---

## SSOT 상태 정의

| 상태 | 의미 |
|------|------|
| Draft | 초안 — 아직 검토 전 |
| Review | 검토 중 |
| Confirmed | CEO 확정 — 절대 기준 |

## 운영 원칙
