# DreamTown Alpha 결정: 단일 별 구조 유지

**확정일**: 2026-03-12 (루미 → CEO 승인)

---

## 결정

Closed Alpha는 현재 단일 별 프론트 구조로 진행한다.
백엔드의 다중 별 구조는 유지하되, `/my-stars` 목록 UX는 다음 스프린트로 이월한다.

---

## Alpha 검증 목표 (이번)

1. 첫 번째 별 생성 경험의 감정 반응
2. 재방문 의사
3. 두 번째 별 생성 니즈 존재 여부

---

## 구현 보류 항목 (다음 스프린트)

- `/my-stars` 목록 페이지
- 사용자 기준 전체 별 조회 UX
- 탭바 다중 별 진입 구조

---

## 현재 구조 (Alpha 유지)

| 계층 | 상태 |
|------|------|
| DB | 다중 별 준비 완료 (user_id 기준, unique 제약 없음) |
| 프론트 | 단일 별 (`/my-star/:id`) 구조 유지 |
| localStorage | `dt_star_id` 1개 저장 유지 |
| 탭바 | 현재 구조 유지 |

---

## 판단 근거

1. 지금 Prototype 검증 목표는 "첫 번째 별 경험"이다.
2. 다중 별 UX 변경은 검증 범위보다 변경 범위가 더 크다.
3. 백엔드는 이미 다중 별을 받을 준비가 되어 있어 제품 리스크가 낮다.

**지금은 첫 별을 아름답게 만들고, 두 번째 별은 사용자가 먼저 원하게 두는 게 맞다.**

---

## Alpha 관찰 신호

```text
사용자가 "두 번째 소원도 만들고 싶다"고 느끼는가
```

이 니즈가 실제로 나오면, 다음 스프린트의 `/my-stars` 목록 페이지는
기능 추가가 아니라 검증된 확장이 된다.
