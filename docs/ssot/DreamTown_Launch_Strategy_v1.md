## 핵심 전략

```
많이 모으기          ❌
맞는 사람에게 깊게 닿기  ✅
```

---

## 1. 첫 런칭 포지셔닝

> **소원을 별로 만드는 조용한 위로 앱**

설명 문장:
> 하루의 조용한 순간, 작은 소원을 별로 남기고 다시 그 별을 보러 오는 서비스

**초기에 전면에 내세우지 않는 것:** 커뮤니티 / 여행 / Universe 확장 / IP

초기 핵심 경험만:
```
소원 → 별 생성 → 내 별 보기
```

---

## 2. 첫 사용자 타깃

> **조금 흔들리지만 아직 포기하지 않은 사람**

- 혼자 조용한 시간을 좋아하는 사람
- 마음이 조금 지친 사람
- 거창한 조언보다 작은 위로를 원하는 사람
- 밤에 자기 마음을 정리하고 싶은 사람

첫 타깃은 **대중 전체가 아니라 정서적으로 맞는 작은 집단**이다.

---

## 3. 핵심 메시지 (3개만)

```
1. 당신의 소원은 혼자가 아닙니다.
2. 작은 소원도 별이 됩니다.
3. DreamTown은 소원을 별로 남기는 조용한 위로의 공간입니다.
```

이 이상 메시지를 늘리지 않는다.

---

## 4. 첫 사용자 경험 — 30초~2분

```
소원 입력 → 별 탄생 → 내 별 보기
```

첫 런칭 목표:
- "와, 이거 예쁘다" ❌
- "이거 조용히 위로된다" ✅

---

## 5. 런치 4단계

| 단계 | 대상 | 목표 |
|------|------|------|
| Stage 1 Closed Prototype | 지인 5~20명 | 흐름 + 감정 검증 |
| Stage 2 Quiet Beta | 정서적으로 맞는 첫 사용자 50~200명 | My Star 재방문 확인 |
| Stage 3 Soft Launch | 소규모 커뮤니티 공개 | Star Share 자연 발생 확인 |
| Stage 4 Public Launch | 전체 공개 | 첫 1,000별 생성 |

---

## 6. 채널 전략

### 우선 채널
- 인스타그램 이미지/릴스
- 카카오 채널 (@dailymiracles)
- 조용한 감성 커뮤니티
- 개인 네트워크 공유

### 후순위 (Public Launch 이후)
- 대규모 광고
- 퍼포먼스 마케팅

## 7. 런칭 핵심 자산 5개

| # | 자산 | 내용 |
|---|------|------|
| 1 | 1문장 소개 | "DreamTown은 소원을 별로 만드는 조용한 위로 앱입니다." |
| 2 | 10초 데모 | 소원 입력 → 별 탄생 → 내 별 보기 |
| 3 | 대표 화면 3장 | Wish Gate / Star Birth / My Star |
| 4 | Star Card | 공유용 대표 카드 1종 |
| 5 | 짧은 소개글 | 감성 설명문 (100자 이내) |

---

## 8. CTA 원칙

**추천 CTA:**
- "당신의 소원을 별로 만들어보세요."
- 내 별 만들기 / 소원을 별로 보내기

**비추천 CTA:**
- 지금 가입하기 / 기능 살펴보기

DreamTown은 **행동보다 감정 진입**이 중요하다.

---

## 9. 핵심 KPI

| 지표 | 기준 |
|------|------|
| 별 생성 수 | 얼마나 많이 만들었는가 |
| My Star 재방문 | 다시 별을 보러 왔는가 ← **핵심** |
| 첫 세션 완료율 | Wish Gate → My Star 완주 비율 |

> **얼마나 많이 들어왔는가보다 얼마나 많이 다시 왔는가**

---

## 10. 런칭 톤 원칙

**해야 하는 것:**
- 짧고 따뜻한 문장
- 밤, 별, 조용함
- 과장 없는 소개

**하지 않는 것:**
- "인생이 바뀝니다"
- "모두를 위한 앱입니다"
- 감성 과장 카피

---

## 11. Launch 핵심 문장

```
DreamTown은 혼자 있는 시간에 작은 소원을 별로 만들며
조용한 위로를 얻는 서비스입니다.
```

```
여수 바다에서 시작된 작은 소원이, 당신의 별이 되는 곳.
```
