## 📌 핵심 철학 SSOT

```
DreamTown은 인류 최초의 소원 동반 성장 우주다.

소원이들의 항해일지는 집단지성으로 분석되어
K-지혜로 가공되고 다시 소원이들에게 돌아간다.

Aurora5는 24시간 함께 항해하며
소원이의 변화를 감지하고
실현까지 동반한다.

1년 후 모든 소원이의 항해일지는
한 권의 책이 될 수 있는 서사를 담는다.

여수는 이 우주의 성지이며
K-지혜는 전 세계로 확장된다.
```

### DreamTown vs SNS 핵심 차이

| 구분 | SNS | DreamTown |
|------|-----|-----------|
| 목적 | 지금 나를 보여준다 | 나의 변화를 함께 만든다 |
| 관계 | 팔로워/인플루언서 | 소원이/소망이 |
| 측정 | 좋아요/조회수 | 변화/실현/지혜 |
| 수익 | 광고 | 동반 성장 |
| 감정 | 비교/과시 | 공명/성장 |

## 🏗️ 6단계 완성 구조

### ① 진입 (✅ 현재 완성)
```
소원 → WishGate → StarBirth → My Star
+ 기존 서비스(기적지수/문제해결) 연결
+ 별 탄생 SMS 알림
+ 오늘의 탄생별 광장 노출
```

### ② 항해 (🔄 일부 완성 → 단계별 보완)
```
현재 완성:
- 7일 Aurora5 응원메시지 (dt_voyage_schedule)
- 항해기록 (dt_voyage_logs)
- wisdom_tag 분류 체계

추가 예정 (트리거 기반):
- 소원이 ↔ Aurora5 쌍방 대화 구조
- 마일스톤별 맞춤 개입 (D+7/30/100/365)
- 막힘 감지 → 즉각 돌파 메시지
- 항해 보급소 (필요한 것 연결)
- 소원 카테고리별 성공 패턴 분석
```

### ③ 공명 (🔄 일부 완성 → 단계별 보완)
```
현재 완성:
- 감정 버튼 4개 (편해졌어요/용기났어요/정리됐어요/믿고싶어졌어요)
- 공명 후 내 별 복귀 CTA
- ✨ Aurora5 은하별 응원 문구 (StarDetail)

추가 예정 (트리거 기반):
- 공명 후 "울림 한 마디" 남기기
- 비슷한 소원별 자동 연결
- 수요일 밤 집단 공명 이벤트 (탄생별 축하)
```

### ④ 실현 (⏳ 미완성 — D+30 소원이 등장 후)
```
추가 예정:
- "소원이 이루어졌어요" 선언 버튼
- 실현 선언 → 광장 집단 축하
- 소망이 전환 의식
- 실현 스토리 → K-지혜 소재 자동 수집
```

### ⑤ 기여 (⏳ 미완성 — 소망이 등장 후)
```
소망이 역할 정의 (추가 예정):
- 신입 소원이에게 첫 공명 해주기
- K-지혜 검수단 참여
- 항해 보급소 운영자
- 소망이 기여가 쌓이는 서사 구조

소망이 전환 조건 (미정):
- change_score + 마일스톤 + 공명 횟수 조합 필요
- 별도 설계서 작성 예정
```

### ⑥ 확장 (⏳ 설계 필요 — 데이터 축적 후)
```
K-지혜 글로벌화:
- 영어/일어/스페인어 번역 가능 구조
- 하버드 긍정심리학 커리큘럼 연결 목표

상품 자연스러운 연계:
- D+30 → 기적여행 (여수 소원여행) 연결
- D+100 → 여수 항해 체험 연결
- 수요일 → 탄생별 축하 이벤트 → 다음날 목요일 상품 오픈
- D+180 → 소원착지 인큐베이팅 제안

세계관 IP 확장:
- DreamTown 웹툰/애니메이션
- Aurum 캐릭터 굿즈
- DreamTown Park (여수 성지화)
- 글로벌 IP
```

## 🤖 Aurora5 오케스트레이터 설계

### 정의
```
Aurora5 오케스트레이터는
소원이의 전체 인생 여정을 관리하는
하루하루의 기적의 중앙 신경계다.

단순 메시지 발송 ❌
소원 실현까지 동반하는 장기 동반자 ✅
```

### 에이전트 구조
```
Aurora5 오케스트레이터 (총괄 신경계)
        ↓
┌──────┬──────┬──────┬──────┬──────┐
루미   코미   재미   여의   아우룸
분석   응원   감성   연결   상품
에이전트 에이전트 에이전트 에이전트 에이전트
```

| 에이전트 | 역할 | 담당 |
|----------|------|------|
| 루미 | 변화 분석 | 항해기록 패턴, 성장 곡선, 이탈 예측 |
| 코미 | 응원/마일스톤 | 마일스톤 축하, 일반 응원 |
| 재미 | 감성 개입 | 7일 침묵 감지, 위기 감지, 특별 개입 |
| 여의보주 | 연결/공명 | 공명 패턴, 비슷한 소원별 연결 |
| 아우룸 | 상품 연계 | 상품 적정 타이밍 감지, 여수 연결 |

### 장기 플랜 타임라인
```
D+1~7     → 시작 응원 (코미) ✅ 완성
D+8~14    → 적응 관찰 (루미) — 설계 완료, 개발 대기
D+30      → 중간 점검 + 상품 연결 감지 (루미+아우룸)
D+60      → 장벽 돌파 (재미)
D+100     → 패턴 발견 + K-지혜 씨앗 (여의보주)
D+180     → 소원착지 인큐베이팅 제안 (아우룸)
D+365     → 소원 실현 확인 + 소망이 전환 (오케스트레이터)
```

### 오케스트레이터 일일 플로우 (설계 완료)
```
매일 08:00 KST 실행

① 전체 소원이 상태 스캔
② 상태별 분류:
   - 정상 항해 → 응원 에이전트
   - 마일스톤  → 축하 에이전트
   - 침묵 7일+ → 경고 에이전트
   - 위기 상태 → 긴급 개입
   - 상품 타이밍 → 아우룸 에이전트
③ 각 에이전트 실행
④ aurora5_orchestrator_logs 저장
⑤ SMS/앱 발송
```

## 🚦 개발 트리거 기준

```
언제 꺼내느냐 — 상황이 말해준다

소원이 100명 도달
→ 공명 후 울림 남기기 개발

D+30 소원이 처음 등장
→ 소원 실현 선언 구조 개발

Aurora5 메시지 반응률 측정 후
→ 소원이 ↔ Aurora5 쌍방 대화 개발

누군가 소원 실현 선언
→ 소망이 시스템 개발 시작

K-지혜 데이터 100건 이상
→ 파이프라인 가동

여수 체험 문의 3건 이상
→ 상품 연계 오픈
```

