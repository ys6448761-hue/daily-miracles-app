## Vision 3단 구조

| 층 | 이름 | 정체성 | 내용 |
|---|------|--------|------|
| **Layer 1** | Product | 조용한 위로 앱 | 소원 → 별 → 내 별 보기 |
| **Layer 2** | Platform | 별 기반 커뮤니티 | 별 → 별자리 → 은하 + Star Share |
| **Layer 3** | Universe | DreamTown Universe | 위로 + 세계관 IP + 여행 + 콘텐츠 |

셋은 충돌하지 않는다. **같은 나무의 다른 성장 단계**이다.

### Layer 1 — Product (현재)
핵심 기능: Wish Gate / Star Birth / My Star / Today Message
핵심 감정: 조용한 위로

### Layer 2 — Platform (중기)
핵심 구조: 별 → 별자리 → 은하
주요 기능: Galaxy Map / Star Growth / Star Share / Community Stars

### Layer 3 — Universe (장기)
확장 영역: 애니메이션 / 웹툰 / 캐릭터 IP / 여수 여행 프로그램 / DreamTown 이벤트

## 성장 단계 (Stage Map)

```
Stage 1   위로 앱
          소원을 별로 만드는 조용한 위로 경험
              ↓
Stage 2   별 루틴 서비스
          My Star 중심 Daily Loop 정착
              ↓
Stage 3   은하 탐험 세계관
          Galaxy Map + 별자리 연결
              ↓
Stage 4   커뮤니티 / 공유
          Star Share + 소망이 나눔
              ↓
Stage 5   여행 / IP / 콘텐츠
          여수 여행 연계 + 캐릭터 IP 확장
              ↓
Stage 6   DreamTown Universe
          위로 + 세계관 + 여행 + 커뮤니티 통합
```

## 사용자 입구 원칙

사람은 처음부터 "거대한 우주"로 들어오지 않는다.
처음엔 아주 작은 입구가 필요하다.

```
입구:  내 소원을 별로 만드는 조용한 위로 경험
확장:  별 → 은하 → Universe
```

**제품 입구는 앱, 확장은 Universe.**

---

## DreamTown이 Universe가 될 수 있는 이유

지금 이미 구조가 Universe 설계이다:

| 요소 | 의미 |
|------|------|
| 여수 Origin | 현실 세계 연결점 |
| Golden Turtle | 신화 상징 |
| 아우룸 | 세계관 안내자 |
| 별 → 별자리 → 은하 | IP 확장 구조 |
| My Star Core Action | 사용자 귀환 루프 |
| Galaxy Evolution | 세계관 성장 엔진 |

이건 기능 설계가 아니라 **Universe 설계**이다.
