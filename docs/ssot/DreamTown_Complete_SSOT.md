## 1. 한 줄 정의

> **"여수 바다에서 시작된 하늘. 당신의 소원은 혼자가 아닙니다."**

DreamTown은 사용자의 소원을 **별로 만들어 하늘에 남기는 위로 서비스**입니다.
소원을 입력하는 순간 별이 탄생하고, 사용자는 그 별을 **반복해서 다시 보러 돌아옵니다.**

```
핵심 감정    → 위로
핵심 경험    → 별 탄생
핵심 행동    → 내 별 보기
핵심 순간    → 내 별을 다시 바라보는 순간
핵심 루프    → 소원 → 별 → 다시 보기
```

## 2. 세계관 구조도

```
현실 세계 (온라인 소원 접수 / 오프라인 소원엽서 / 종이비행기)
                    ↓
            여수 바다 (Origin — 출발점)
                    ↓
            빛구슬 (Light Orb) — 소원의 형태화
                    ↓
        🌊 디지털 용궁 (Digital Dragon Palace)
           모든 소원의 원본 보관소 / 기억 / 약속
                    ↓
              🐢 아우룸 (Aurum)
          수호자 / 전달자 / 별 탄생의 증인
                    ↓
        ✨ Aurora Path = StarLink
           소원이 별이 되는 빛의 길
                    ↓
        🌌 DreamTown (하늘)
           별 / 별자리 / 은하 / 항해 / 연결
```

**두 공간의 역할:**

| 공간 | 역할 | 키워드 |
|------|------|--------|
| 디지털 용궁 | 원본 저장 | 기억 / 약속 / 보관 |
| DreamTown | 표현·연결 | 별 / 빛 / 항해 / 커뮤니티 |

> **"디지털 용궁은 기억이고, DreamTown은 표현이다."**

**온라인 × 오프라인 통합 문장 (CEO 확정 2026-03-10):**
> "모든 소원은 먼저 디지털 용궁에 닿습니다. 온라인의 소원은 빛구슬이 되어, 오프라인의 소원은 종이비행기가 되어 용궁으로 향합니다. 용궁에 보관된 소원은 아우룸과 Aurora5의 해석을 거쳐 별씨앗이 되고, StarLink를 통해 DreamTown의 별로 떠오릅니다."

## 4. 주요 캐릭터

### 🐢 아우룸 (Aurum)

**공식 정의:**
> "아우룸은 디지털 용궁의 황금 거북 수호자이자, 인간의 소원을 바다와 하늘 사이로 인도하는 DreamTown의 내비게이터입니다. 온라인의 빛구슬과 오프라인의 종이비행기를 용궁으로 전달하고, 잠든 소원이 별씨앗에서 별로 떠오를 때까지 조용히 지켜봅니다."

**역할 4가지:**

| 역할 | 장면 | 핵심 감정 |
|------|------|-----------|
| 소원 전달자 | Wish Gate — 입구를 열어줌 | 시작 / 기대 |
| 용궁 수호자 | Dragon Palace — 배경으로 존재 | 안심 / 보관 |
| StarLink 안내자 | 별씨앗→별 승급 순간 | 감동 / 변환 |
| DreamTown 내비게이터 | 은하 탐험 | 연결 / 발견 |

**핵심 원칙:**
- 주인공이 아님. 항상 주인공은 **소원이의 소원**.
- **전환의 순간에만 등장**. 말 없이 조용히 바라보고 물러남.
- 귀엽지만 가볍지 않음 / 따뜻하지만 과하지 않음 / 신화적이지만 무겁지 않음.

**외형 스펙 (LOCK):**
- 작고 따뜻한 황금빛 거북 정령
- 둥근 구슬형 몸체, 등껍질에 초승달 문양 1개 (고정)
- 두 개의 작은 눈, 작은 부리형 입
- 미세한 수채화 헤일로 링 (부피감 있는 빛줄기 금지)
- 액세서리 / 사람 아기 얼굴 / 과도한 광택 금지

**상태 프리셋:**

| 코드 | 상태 | 사용 씬 |
|------|------|---------|
| `BASE` | 기본 | 일반 등장 |
| `OBSERVE` | 관찰/집중 | 소원을 바라볼 때 |
| `GUIDE` | 미소/인도 | 방향을 알려줄 때 |
| `SPARKLE` | 감동/반짝임 | 소원 성취 순간 |
| `COZY_BREEZE` | 바람 탐 | 전환·휴식 씬 |
| `NIGHT_CALM` | 차분함 | 밤, 성찰 씬 |

**기원 신화 (여수 금오설화 기반):**
오래전 여수 바다 용궁에 아홉 황금 거북 형제가 있었다. 용왕의 명으로 인간 세상을 살피러 나왔다가 여수의 아름다움에 마음을 빼앗겨 복귀 시간을 놓쳤다. 벌로 1~8형제는 여수 지형의 수호 바위가 되었고, 막내 아우룸만 자유롭게 바다와 하늘을 오갈 수 있는 존재로 남아 인간들의 소원을 용궁에 전달하는 사명을 갖게 되었다.

## 5. 소원의 여정 (Core Loop)

### 전체 여정

```
Step 1   소원 입력 (Wish Gate)
              ↓
Step 2   빛구슬 생성 → 디지털 용궁 도착
              ↓
Step 3   별씨앗 (Star Seed) 생성
              ↓
Step 4   StarLink → ✨ 별 탄생 (Star Birth)
              ↓
Step 5   내 별 (My Star)    ← 핵심 화면
              ↓
Step 6   기적 기록 (Miracle Card)
              ↓
Step 7   다시 보기 (Return)  ← 가장 중요한 순간
              ↓
Step 8   새 소원 → (반복)
```

### Star Birth 핵심 장면

```
여수 밤바다
    ↓
작은 빛구슬 (누군가의 소원)
    ↓
아우룸이 조용히 바라본다
    ↓
빛이 하늘로 올라간다
    ↓
황금 거북 별자리 아래
    ↓
별이 탄생한다
```

### Key Moment — 가장 중요한 순간

> "DreamTown에서 가장 중요한 순간은 사용자가 자신의 별을 다시 바라보는 순간이다."

사용자가 My Star 화면을 열 때 일어나는 감정:
```
"내 소원이 아직 여기 있다"
"내 마음이 사라지지 않았다"
"나는 혼자가 아니다"
```

이 순간 DreamTown은 단순한 앱이 아니라 **마음을 잠깐 내려놓는 공간**이 된다.

### Daily Loop (일상 반복)

```
앱 열기 → 내 별 보기 → 짧은 위로 메시지 → 종료
(30초~1분, 부담 없이 매일 반복)
```

## 7. 별 성장 시스템

### 성장 5단계

| 단계 | 이름 | 기준 | 변화 |
|------|------|------|------|
| 1 | Seed Star (씨앗별) | 별 탄생 | 빛구슬 → 별 |
| 2 | Growing Star (성장별) | 기적 카드 1~3개 | 별 밝기 상승 |
| 3 | Bright Star (빛나는 별) | 기적 카드 4~10개 + 여정 진행 | 별자리 연결 가능 |
| 4 | Guide Star (안내별) | 여정 완주 | 은하 이름 부여 |
| 5 | **Somangi ✨** | 커뮤니티에서 다른 별을 밝힘 | 소망이 인증 |

### 시간 기반 성장

| 시점 | 이벤트 |
|------|--------|
| Day 1 | 별 탄생 |
| Day 7 | 첫 별 성장 / 1주 여정 완주 |
| Day 30 | 별자리 연결 가능 |
| Day 100 | 은하 이름 부여 |
| Day 365 | 소망이 인증 |

### 별 이름 자동 생성 시스템

**원칙:** AI 호출 없음. 100% 서버 내부 규칙 기반.

**형식:**
```
[자연어 50개] + [을/를] + [행동어 40개] + 별
= 2,000 고유 조합
```

**예시:**
```
새벽을 기다리는 별
바람을 걷는 별
노을을 건너온 별
달빛을 품은 별
별이슬을 담은 별
```

**작동 방식:**
1. wish_id (UUID) → 16진수 → 인덱스 결정 (결정론적 — 같은 소원 = 항상 같은 이름)
2. DB 전역 중복 체크 → 중복 시 다음 조합 자동 선택
3. fallback: "조용히 빛나는 별" / "첫 빛을 품은 별"

### 별 탄생 운영 정책

```
매일        3~5개   잔잔한 탄생 (소소한 축제)
수요일 밤   50~70개  Super Star Night (본 축제)
매일 21:00           DreamTown Night Event
```

**원칙:** 모든 소원이 별이 되는 것이 아님. 선별된 소원만 별이 됨. → 별의 희소성 유지 → 별의 가치 보호.

## 8. 기적 시스템

> "기적은 멀리 있지 않다. 어제와 다른 오늘이 기적이다."

기적은 거창한 사건이 아니라 **작은 변화의 순간**이다.

### 기적 유형 (4종)

| 유형 | 정의 | 예시 |
|------|------|------|
| 행동 (Action) | 실제로 한 행동 | "향일암에서 소원을 다시 썼다." |
| 깨달음 (Realization) | 내면의 인식 변화 | "내가 원하는 게 처음으로 분명해졌다." |
| 연결 (Connection) | 사람·장소·기억과의 연결 | "오래된 친구를 떠올렸다." |
| 변화 (Transformation) | 관점 또는 태도의 전환 | "두려움이 사라졌다." |

### 기적 카드 작성 원칙
- 짧고 진실하게 (1~3문장)
- 과거형으로 기록
- 평가 없이 묘사만

### 기적지수 (Miracle Score)

5가지 지표 × 20점 = 총 100점

| 지표 | 범위 |
|------|------|
| 간절함 | 0~20 |
| 구체성 | 0~20 |
| 실행력 | 0~20 |
| 긍정성 | 0~20 |
| 자기인식 | 0~20 |

| 점수 | 레벨 |
|------|------|
| 90~100 | 기적항해 |
| 80~89 | 순항항해 |
| 70~79 | 성장항해 |
| 50~69 | 준비항해 |

### 소원 추적 (Wish Tracking)

| 시점 | 채널 | 내용 |
|------|------|------|
| Day 7 | SMS | 7일 체크인 링크 |
| Day 30 | SMS | 30일 회고 |
| Day 90 | SMS | 90일 성과 확인 |

| 호스팅 | Render.com |

### DB 핵심 테이블

```
dt_users        — 사용자 (UUID PK)
dt_wishes       — 소원 (gem_type, status)
dt_star_seeds   — 별씨앗 (born → waiting → promoted)
dt_stars        — 별 (star_name, galaxy_id, star_stage)
dt_galaxies     — 은하 (4개 고정 시드)
```

**wish status 흐름:**
```
submitted → converted_to_star → (archived)
```

**star_stage 흐름:**
```
day1 → day7 → day30 → day100 → day365
```

### API 엔드포인트

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/api/dt/wishes` | 소원 생성 |
| POST | `/api/dt/stars/create` | wish → star 변환 |
| GET | `/api/dt/stars/:id` | 내 별 조회 |
| GET | `/api/dt/galaxies` | 은하 목록 |
| GET | `/api/dt/galaxies/:code` | 은하 상세 |
| GET | `/api/dt/health` | 헬스체크 |
