# 딜 등록 및 보호 정책

> MICE 인센티브 딜 관리 지침 (증빙 중심)
> 버전: 1.0 | 2026-02-05

---

## 1. 딜 등록 절차

### 1.1 등록 요건

| 단계 | 요건 | 증빙 |
|------|------|------|
| **예비 딜** | 행사 문의 접수 | 이메일/문서 접수 기록 |
| **확정 딜** | 계약서 체결 | 계약서 사본 |
| **진행 딜** | 선급금 입금 | 입금 확인서 |
| **완료 딜** | 행사 종료 + 정산 | 결과보고서 제출 |

### 1.2 등록 필수 정보

```
- 행사명 (공식 명칭)
- 행사 기간 (시작일 ~ 종료일)
- 행사 장소 (여수시 내)
- 예상 규모 (참가인원, 외국인 비율)
- 주최/주관 기관명
- 담당자 연락처
- 예상 지원 금액
```

## 2. 딜 보호 정책

### 2.1 보호 기간

| 딜 상태 | 보호 기간 | 연장 조건 |
|---------|----------|----------|
| 예비 딜 | 30일 | 진행 상황 보고 시 30일 연장 |
| 확정 딜 | 행사 종료 + 30일 | 결과보고 제출까지 |
| 진행 딜 | 행사 종료 + 60일 | 정산 완료까지 |

### 2.2 보호 범위

**보호 대상**
- 동일 행사에 대한 타 업체 제안 차단
- 동일 주최자의 유사 행사 (6개월 내)
- 동일 기간 동일 장소 행사

**보호 제외**
- 주최자가 직접 다른 업체 지정 요청
- 6개월 이상 진행 없는 딜
- 증빙 불충분 딜

### 2.3 분쟁 해결

| 상황 | 판단 기준 | 우선 순위 |
|------|----------|----------|
| 동시 등록 | 등록 시간 | 먼저 등록한 쪽 |
| 증빙 충돌 | 증빙 강도 | 계약서 > 이메일 > 구두 |
| 주최자 변심 | 서면 통보 | 서면 통보 기준 |

## 3. 증빙 체계

### 3.1 증빙 등급

| 등급 | 증빙 유형 | 효력 |
|------|----------|------|
| **A급** | 계약서, 세금계산서, 입금증 | 완전 증빙 |
| **B급** | 공문, 견적서, 이메일 (공식) | 유효 증빙 |
| **C급** | 카톡/문자, 구두 확인 | 보조 증빙 |

### 3.2 등급별 인정 범위

| 딜 단계 | 최소 요구 등급 |
|---------|---------------|
| 예비 딜 | C급 이상 |
| 확정 딜 | B급 이상 |
| 진행 딜 | A급 필수 |
| 정산 신청 | A급 100% |

### 3.3 증빙 보관 규칙

```
파일명 규칙: [YYYYMMDD]_[유형]_[행사명]_[순번].[확장자]
예: 20260815_계약서_여수MICE대회_01.pdf

보관 기간: 행사 종료 후 5년
보관 위치: ops-center 에셋 저장소 + 로컬 백업
```

## 4. MICE 인센티브 증빙 요건

### 4.1 결과보고 필수 증빙

| 항목 | 증빙 | 비고 |
|------|------|------|
| 참가자 등록부 | 명단 + 입금 확인 | 사전등록: 입금일 필수 |
| 숙박확인서 | 세금계산서/영수증 | 객실수, 숙박일 명시 |
| 지출증빙 | 세금계산서 + 입금증 | 간이영수증 불인정 |
| 사진대장 | 로고 노출 2장+ | 고화질 권장 |
| 통장사본 | 인센티브 입금 계좌 | 최근 거래내역 포함 |

### 4.2 결제수단별 요건

| 결제수단 | 필수 서류 | 주의사항 |
|----------|----------|----------|
| **계좌이체** | 세금계산서 + 입금증 + 사업자등록증 | 입금일 일치 확인 |
| **법인카드** | 세금계산서 + 카드전표 | 법인명 일치 확인 |
| **현금** | 세금계산서 + 입금증 | 10만원 이상 현금영수증 |

### 4.3 불인정 항목

❌ **지원 불가**
- 간이영수증
- 개인카드 결제
- 현금 (증빙 없음)
- 행사 기간 외 지출
- 여수시 외 지역 지출 (일부 항목)

## 5. 딜 현황 관리

### 5.1 상태 코드

| 상태 | 코드 | 설명 |
|------|------|------|
| 문의 | INQUIRY | 초기 문의 단계 |
| 제안 | PROPOSAL | 견적/제안 발송 |
| 협의 | NEGOTIATION | 조건 협의 중 |
| 계약 | CONTRACTED | 계약 체결 완료 |
| 진행 | IN_PROGRESS | 행사 준비/진행 중 |
| 완료 | COMPLETED | 행사 종료 |
| 정산 | SETTLEMENT | 인센티브 정산 중 |
| 종결 | CLOSED | 모든 처리 완료 |
| 취소 | CANCELLED | 취소/철회 |

### 5.2 전환 조건

```
INQUIRY → PROPOSAL: 견적서 발송
PROPOSAL → NEGOTIATION: 주최자 피드백
NEGOTIATION → CONTRACTED: 계약서 체결
CONTRACTED → IN_PROGRESS: 선급금 입금
IN_PROGRESS → COMPLETED: 행사 종료
COMPLETED → SETTLEMENT: 결과보고 제출
SETTLEMENT → CLOSED: 인센티브 입금 확인
```

## 7. 분쟁 사례 및 판례

### 사례 1: 동시 접촉
```
상황: A업체와 B업체가 동일 주최자에게 같은 주 접촉
판단: 먼저 이메일 발송한 A업체 우선
증빙: 이메일 발송 시간 (A: 월요일 09:00, B: 화요일 14:00)
```

### 사례 2: 주최자 변심
```
상황: A업체와 계약 후 주최자가 B업체로 변경 요청
판단: A업체 보호 기간 적용, 위약금 협의
증빙: 계약서 + 변경 요청 공문
```

### 사례 3: 증빙 분쟁
```
상황: A업체는 구두 확인, B업체는 이메일 증빙
판단: B급 증빙(이메일)이 C급(구두)보다 우선
결과: B업체 딜 인정
```
