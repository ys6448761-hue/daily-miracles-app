### 2.2 Wix 후기 폼 (웹)

```
URL: /review 또는 /후기

폼 필드:
├── 닉네임 (필수, 공개용)
├── 소원 카테고리 (선택)
├── 별점 (1-5, 필수)
├── 한줄 후기 (필수, 50자 이내)
├── 상세 후기 (선택, 500자 이내)
├── 사진 첨부 (선택)
└── 공개 동의 (필수)
```

### 2.3 카카오채널 1:1 채팅 (수동)

```
재미(CRO)가 직접 수집
→ 감동적인 후기는 스토리화
→ 소원이 동의 하에 콘텐츠화
```

## 3. 후기 질문 설계

### 3.1 Day 7 핵심 후기 (간단)

```
Q1. 응원 메시지가 도움이 되셨나요?
    ○ 매우 도움됨 ⭐⭐⭐⭐⭐
    ○ 도움됨 ⭐⭐⭐⭐
    ○ 보통 ⭐⭐⭐
    ○ 별로 ⭐⭐
    ○ 전혀 ⭐

Q2. 한줄로 표현해주세요 (50자)
    예: "매일 아침 메시지가 출근길 힘이 됐어요"

Q3. 가장 좋았던 점은? (택1)
    ○ 매일 오는 응원 메시지
    ○ 예쁜 소원그림
    ○ 따뜻한 말투
    ○ 나만을 위한 맞춤 응원
    ○ 기타: _______
```

### 3.2 성공 스토리 (상세)

```
Q1. 어떤 소원을 빌었나요?
    (카테고리: 취업/연애/건강/학업/관계/기타)

Q2. 소원이 이루어졌나요?
    ○ 완전히 이루어짐 🎉
    ○ 부분적으로 이루어짐
    ○ 아직 진행 중
    ○ 다른 방향으로 좋은 결과

Q3. 하루하루의 기적이 어떻게 도움이 됐나요?
    (자유 서술, 200자 이상)

Q4. 다른 소원이들에게 한마디!
    (공개용 메시지)

Q5. 상세 후기 공개에 동의하시나요?
    ○ 닉네임으로 공개 OK
    ○ 익명으로 공개 OK
    ○ 비공개 희망
```

## 4. 데이터 저장 구조

### 4.1 Airtable 테이블: Reviews

| 필드명 | 타입 | 설명 |
|--------|------|------|
| review_id | Auto | 고유 ID |
| wish_id | Link | 연결된 소원 |
| nickname | Text | 공개용 닉네임 |
| category | Select | 소원 카테고리 |
| rating | Number | 별점 (1-5) |
| one_liner | Text | 한줄 후기 |
| detail | Long Text | 상세 후기 |
| best_point | Select | 가장 좋았던 점 |
| photo_url | URL | 첨부 사진 |
| consent | Checkbox | 공개 동의 |
| status | Select | pending/approved/featured |
| collected_at | Date | 수집일 |
| source | Select | kakao/wix/manual |

### 4.2 상태 관리

```
pending (대기)
    │
    ▼ 코미/재미 검토
approved (승인)
    │
    ▼ 특별히 좋은 후기
featured (추천) ← Wix 메인에 노출
```

## 5. 자동화 파이프라인

### 5.1 Day 7 자동 후기 요청

```javascript
// routes/reviewRoutes.js (신규)

// 1. Day 7 되면 자동 트리거
POST /api/reviews/request-day7
{
  wish_id: "xxx",
  phone: "010-xxxx-xxxx",
  name: "소원이"
}

// 2. 카카오 알림톡 발송 (Solapi)
→ 후기 작성 링크 포함

// 3. 링크 클릭 → Wix 폼 or 자체 폼
→ 작성 완료 시 Airtable 저장
```

### 5.2 후기 작성 완료 시

```javascript
// Webhook: 후기 작성 완료

POST /webhooks/review-submitted
{
  review_id: "xxx",
  rating: 5,
  one_liner: "매일 힘이 됐어요"
}

// 자동 처리:
// 1. 감사 메시지 발송
// 2. 별점 4-5점 → 코미에게 알림 (featured 후보)
// 3. 통계 업데이트
```

### 5.3 성공 스토리 수집

```javascript
// 수동 트리거 (재미 담당)

POST /api/reviews/request-success-story
{
  wish_id: "xxx",
  trigger: "소원이가 취업 성공 알림"
}

// 1. 성공 스토리 폼 링크 발송
// 2. 작성 완료 시 재미 검토
// 3. 스토리 콘텐츠화 (SNS/Wix)
```

## 6. 후기 활용 계획

### 6.1 Wix 노출 위치

```
[메인 페이지]
├── 후기 슬라이더 (featured 3-5개)
└── "4.8/5.0 ⭐ (127개 후기)"

[팀 소개 페이지]
└── "이 팀이 도와준 소원이들의 이야기"

[후기 전용 페이지] /reviews
├── 전체 후기 목록
├── 카테고리 필터
└── 성공 스토리 하이라이트
```

```
카카오채널:
└── 주간 베스트 후기 공유
```

## 7. 후기 수집 인센티브

### 7.1 기본 감사

```
후기 작성 완료 시:
→ "감사합니다" 메시지 + 특별 응원 카드 1장
```

### 7.2 상세 후기 보너스 (선택)

```
100자 이상 상세 후기:
→ 다음 소원 시 "프리미엄 소원그림" 무료 업그레이드

성공 스토리 작성:
→ "기적 서포터" 뱃지 + 1회 무료 이용권
```

### 7.3 사진 후기 보너스

```
소원그림과 함께 찍은 사진:
→ SNS 콘텐츠 활용 시 소정의 기프티콘
```

## 8. 품질 관리

### 8.1 가짜 후기 방지

```
✅ wish_id 연결 필수 (실제 이용자만)
✅ 중복 작성 방지 (1소원 = 1후기)
✅ 코미/재미 수동 검토 후 공개
```

### 8.2 부정 후기 대응

```
별점 1-2점 후기:
1. 즉시 재미(CRO)에게 알림
2. 24시간 내 1:1 연락
3. 문제 해결 시도
4. 해결 후 후기 수정 요청 (강요 X)
```

### 8.3 검토 체크리스트

```
□ 실제 이용자인가?
□ 욕설/비방 없는가?
□ 개인정보 노출 없는가?
□ 공개 동의했는가?
□ 내용이 구체적인가?
```

## 9. 성과 지표 (KPI)

| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 후기 응답률 | 30% 이상 | (후기 수 / Day7 완료 수) |

| 성공 스토리 수 | 월 5건 이상 | featured 후기 수 |
| SNS 활용률 | 월 10건 | 후기 기반 콘텐츠 수 |

## 10. 구현 우선순위

### Phase 1 (즉시)

```
□ Airtable Reviews 테이블 생성
□ Wix 후기 폼 페이지 생성
□ Day 7 수동 후기 요청 (카톡 직접 발송)
```

### Phase 2 (1주 내)

```
□ Day 7 자동 알림톡 연동
□ 후기 작성 완료 웹훅
□ Wix 후기 슬라이더 구현
```

### Phase 3 (2주 내)

```
□ 성공 스토리 수집 프로세스
□ 후기 카드 이미지 템플릿 (재미)
□ SNS 자동 포스팅 연동
```

## 11. 예상 시나리오

### 이상적인 흐름

```
1. 소원이가 소원 등록
2. 7일간 응원 메시지 수신
3. Day 7: 자동 후기 요청 알림톡
4. 소원이가 5점 + "매일 힘이 됐어요" 작성
5. 코미가 검토 → featured 선정
6. Wix 메인 + 인스타에 노출
7. 새로운 소원이가 후기 보고 신뢰 → 가입
8. 선순환 🔄
```

### 성공 스토리 흐름

```
1. 소원이가 "취업 성공!" 카톡으로 알림
2. 재미가 축하 + 성공 스토리 요청
3. 소원이가 상세 스토리 작성
4. 재미가 카드뉴스로 제작
5. 인스타 + Wix에 "실제 기적 사례"로 게시
6. MZ 타겟 "와 진짜네" → 신뢰도 UP
```
