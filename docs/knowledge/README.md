# Knowledge Cards (지식카드)

> RAW 문서에서 승격된 핵심 지식 자산

---

## 폴더 구조

```
docs/knowledge/
├── README.md (이 파일)
├── 2026-01/
│   ├── KC-2026-0122-001.md
│   ├── KC-2026-0122-002.md
│   └── ...
└── 2026-02/
    └── ...
```

## 지식카드란?

- RAW 원본에서 **승격**된 핵심 지식
- 표준 템플릿으로 **구조화**
- 태그로 **분류**되어 검색 용이
- 팀 전체에서 **재사용** 가능

## 승격 기준

1. **재사용 가능성** - 같은 상황에서 다시 참조할 가치
2. **범용성** - 여러 상황에 적용 가능
3. **팀 공유 가치** - 팀 전체에 유용

## 파일명 규칙

```
KC-YYYY-MMDD-###.md

예시:
- KC-2026-0122-001.md (2026년 1월 22일 첫 번째 카드)
- KC-2026-0122-002.md (2026년 1월 22일 두 번째 카드)
```

## 태그 체계

| 필수 태그 | 설명 |
|----------|------|
| #전략 | 방향성, 의사결정, 비전 |
| #기술 | 개발, 시스템, API |
| #운영 | 프로세스, 루틴 |
| #콘텐츠 | 메시지, 디자인, 영상 |
| #분석 | 데이터, 인사이트 |
| #고객 | 소원이, 피드백 |

| 선택 태그 | 설명 |
|----------|------|
| #긴급 | 즉시 참조 필요 |
| #검증됨 | 효과 확인됨 |

## 검색 방법

```bash
# 태그로 검색
grep -r "#전략" docs/knowledge/

# 키워드로 검색
grep -ri "마케팅" docs/knowledge/
```
