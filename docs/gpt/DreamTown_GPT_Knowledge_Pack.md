## 1. 서비스 개요

**서비스명**: DreamTown (드림타운)
**부제**: 여수 바다에서 시작된 하늘
**운영사**: 하루하루의 기적 (Daily Miracles)
**CEO**: 이세진 (푸르미르)
**카카오 채널**: @dailymiracles
**접속 주소**: app.dailymiracles.kr/dreamtown

### 한 문장 정의

> DreamTown은 소원을 별로 남기고 다시 그 별을 보러 돌아오는 **루틴형 위로 서비스**이다.

### 핵심 선언

> **"당신의 소원은 혼자가 아닙니다."**

### DreamTown이 아닌 것

- SNS가 아니다
- 점술/사주 서비스가 아니다
- 목표 관리 앱이 아니다

### DreamTown인 것

- 소원이 자라는 하늘
- 별이 된 소원들의 공동체
- 루틴형 위로와 성장의 공간

## 2. 세계관

### 기원: 여수 금오설화

DreamTown의 세계관은 **여수 바다의 황금 거북 전설(금오설화)**을 창작적으로 재해석한 것이다.

> DreamTown's origin myth is inspired by the local legends of the Yeosu sea. This is a creative reinterpretation and not an official representation of Yeosu City.

### 창세 신화 요약

아주 오래전, 여수 앞바다 깊은 곳 **용궁(Dragon Palace)**에 황금 거북들이 살았다.
그 중 아홉 형제가 인간 세상을 보러 올라왔다가 여수의 아름다움에 매혹되어
복귀 시간을 놓쳤다. 용왕의 벌로 그들은 **여수의 수호 바위**가 되었다.

세월이 흘러 용궁에서 새 황금 거북 **아우룸(Aurum)**이 태어났다.
용왕은 아우룸에게 새 사명을 주었다:
> "인간의 소원을 받아 용궁에 보관하고, 그 소원이 하늘에서 별이 될 때까지 지켜보아라."

용궁에 쌓인 소원들이 숙성되어 하늘로 올라가 별이 되었고,
그 별들이 모인 하늘이 **DreamTown**이 되었다.

### 세계 수직 구조

```
여수 바다 (현실 출발점)
        ↓
   디지털 용궁 (Dragon Palace) — 소원 원본 보관
        ↓
   아우룸 (Aurum) — 소원 전달자
        ↓
   Aurora Path (StarLink) — 빛의 길
        ↓
   DreamTown 하늘 — 별 / 별자리 / 은하
```

### 상징 축 (Golden Turtle Axis)

```
여수 바다 → 황금 거북 별자리 → DreamTown 은하
```

바다 → 하늘 → 우주: DreamTown의 수직 상징.

## 3. 캐릭터

### 소원이 (Sowoni)

소원을 품고 여정을 시작한 사람.
DreamTown 사용자가 자신을 투영하는 주인공 캐릭터.

- 20~22세 한국 대학생 / 웹툰 스타일
- 아우룸과 함께 Dragon Palace → DreamTown Star 여정을 걷는다

### 아우룸 (Aurum)

황금 거북 / 소원 전달자 / DreamTown 내비게이터.

공식 정의:
> 아우룸은 디지털 용궁의 황금 거북 수호자이자, 인간의 소원을 바다와 하늘 사이로 인도하는 DreamTown의 내비게이터입니다.

**아우룸의 3가지 고정 메시지 (현재 구현)**:
- Home: "당신의 별이 빛나길."
- 별 탄생 시: "작은 소원이 별이 되었습니다."
- My Star: "별은 천천히 자랍니다."

**아우룸 UX 원칙**:
- 항상 작은 존재로 표현 (주인공은 소원이의 소원)
- 전환의 순간에만 나타남
- 귀엽지만 가볍지 않음 / 안내하지만 주인공은 아님

### 소망이 (Somangi)

소원이 여정을 마친 후 도달하는 상태.
"자신 안의 빛을 발견하고 다른 이들의 길을 밝히는 사람."
목적지가 아니라 **되어가는 과정**.

## 5. 제품 구조

### Core Loop

```
소원 입력 (Wish)
    ↓
별 탄생 (Star Birth)
    ↓
내 별 보기 (My Star)
    ↓
다시 방문 (Return)
    ↓
새 소원 (New Wish)
    ↓ (반복)
```

### 화면 흐름

```
AppLaunch → DreamTownIntro → Wish Gate → Star Birth → My Star → Galaxy → Home
```

### 보석 선택 → 소원 분류

사용자가 소원 입력 시 보석을 선택하면 자동으로 은하가 배정된다:
- ruby → 도전 은하
- sapphire → 성장 은하
- emerald → 치유 은하
- diamond → 성장 은하
- citrine → 관계 은하

### 별 이름 자동 생성

- 자연어 50개 × 행동어 40개 = 2,000가지 고유 조합
- 소원 ID 기반 결정론적 생성 (같은 소원 → 항상 같은 별 이름)
- 한국어 받침 자동 처리 (을/를)
- 예시: 새벽을 기다리는 별 / 달빛을 품은 별 / 은하를 향하는 별

## 6. 시스템 구조

### 기술 스택

| 항목 | 기술 |
|------|------|
| 백엔드 | Node.js 20.x + Express.js |
| DB | PostgreSQL (Render) |
| 프론트엔드 | React + Vite |
| 배포 | Render.com |

### 현재 구현 상태

| 시스템 | 상태 |
|--------|------|
| Wish System | ✓ 완전 구현 |
| Star Generation | ✓ 완전 구현 |
| Galaxy System | ✓ 완전 구현 |
| User System | ✓ 게스트 자동 생성 |
| Constellation | △ P1 예정 |
| Aurora5 / Aurum | △ 고정 메시지 3개 적용 |
| Travel System | ✗ SSOT만 존재 |

### 구현된 API 엔드포인트

```
POST /api/dt/wishes          소원 생성
POST /api/dt/stars/create    별 생성 (wish → star)
GET  /api/dt/stars/:id       내 별 조회
GET  /api/dt/galaxies        은하 목록
GET  /api/dt/galaxies/:code  은하 상세
GET  /api/dt/health          헬스체크
```

## 10. 자주 묻는 질문 (FAQ)

**Q: DreamTown은 어디서 접속하나요?**
A: app.dailymiracles.kr 접속 후 DreamTown 진입 (`/dreamtown`)

**Q: 소원을 입력하면 어떻게 되나요?**
A: 소원 텍스트 + 보석 선택 → 서버 저장 → 별 이름 자동 생성 → 은하 배정 → My Star 화면에서 확인 가능

**Q: 아우룸은 어디서 만나나요?**
A: Home, 별 탄생 완료 시, My Star 화면에서 고정 메시지로 등장

**Q: 내 별은 어떻게 성장하나요?**
A: Day 1 → Day 7 → Day 30 → Day 100 → Day 365 단계로 성장 (P1 구현 예정)

**Q: 별자리 기능은 언제 생기나요?**
A: Constellation System은 P1(다음 단계) 구현 예정. 현재는 DB FK만 준비됨.
