# Part 5: self-checker 스킬

## 목적

> 작업 완료 후 누락 없이 완료했는지 스스로 검증하고
> 개선 가능한 부분을 조언한다.

---

## 실행 시점

- 모든 작업 완료 직전
- "완료 보고" 전에 반드시 실행

---

## 점검 항목 (4단계)

### 1. 지시서 완료 검증

| 점검 | 확인 방법 |
|------|----------|
| 모든 Phase 실행? | 지시서의 Phase 목록과 완료 목록 대조 |
| 각 Phase 항목 처리? | Phase별 체크리스트 확인 |
| 파일 생성 완료? | 지시서에서 파일 경로 추출 → 존재 확인 |
| 파일 수정 완료? | 수정 대상 파일 목록 대조 |

### 2. 파일 존재 검증

```bash
# 각 파일 경로에 대해
ls -la [파일 경로]

# 확인 사항
- 파일 존재 여부
- 파일 크기 (0 bytes = 빈 파일)
```

### 3. 내용 검증

| 파일 유형 | 필수 섹션 |
|----------|----------|
| 에이전트 (.md) | 역할, 지침, 자가 검증, 피드백 기록 |
| 스킬 (.md) | 개요, 기능, 사용법/예시 |
| 문서 (.md) | YAML frontmatter, 제목, 본문 |

### 4. 보완 조언 생성

- 추가하면 좋을 관련 기능
- 개선할 수 있는 부분
- 연결하면 좋을 다른 구성요소
- 다음 단계 추천

---

## 출력 형식

```markdown
## 자가 점검 결과

### 1. 지시서 완료 현황

| Phase | 상태 | 비고 |
|-------|------|------|
| Phase 1: [이름] | O | 완료 |
| Phase 2: [이름] | X | [누락 사유] |

**완료율: X/Y (Z%)**

### 2. 파일 검증

| 파일 | 존재 | 내용 | 비고 |
|------|------|------|------|
| path/file1.md | O | O | 정상 |
| path/file2.md | X | - | 미생성 |

### 3. 누락 항목

- [ ] [누락된 작업 1]
- [ ] [누락된 작업 2]

### 4. 보완 조언

1. **[조언 1]**: [설명]
2. **[조언 2]**: [설명]

### 5. 최종 판정

| 항목 | 결과 |
|------|------|
| **종합** | **PASS / FAIL** |
```

---

## 실행 흐름

```
1. 작업 완료 직전
      ↓
2. 지시서에서 Phase/파일 목록 추출
      ↓
3. 각 Phase 완료 여부 확인
      ↓
4. 각 파일 존재/내용 검증
      ↓
5. 누락 항목 목록 생성
      ↓
6. 보완 조언 생성
      ↓
7. 최종 판정 (PASS/FAIL)
      ↓
8-A. PASS → 완료 보고 진행
8-B. FAIL → 누락 항목 처리 후 재점검
```

---

## FAIL 시 행동

```
1. 누락 항목 목록 확인
2. 누락 항목 처리 (파일 생성/내용 추가)
3. self-checker 재실행
4. PASS될 때까지 반복 (최대 3회)
5. 3회 초과 시 → 사용자에게 확인 요청
```

## comi-orchestrator 연동

comi-orchestrator에 "작업 완료 프로토콜" 섹션 추가:

```markdown
## 작업 완료 프로토콜 (필수!)

모든 작업은 다음 단계를 거쳐야만 완료로 인정됩니다:

1. 작업 실행 완료
2. self-checker 실행 (필수!)
3. 자가 점검 결과 확인
4-A. PASS → 완료 보고
4-B. FAIL → 누락 항목 처리 → 2번으로 돌아감
```

## 사용 예시

### 자동 실행

```
[작업 완료 후]
→ self-checker 자동 실행
→ PASS/FAIL 판정
→ PASS면 완료 보고
```

### 수동 실행

```
/self-checker 방금 작업 점검해줘
```
