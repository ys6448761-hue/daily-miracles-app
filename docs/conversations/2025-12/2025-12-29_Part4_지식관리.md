## 문서 저장 규칙

### 대화 기록 (conversations/)

| 항목 | 규칙 |
|------|------|
| 파일명 | `YYYY-MM-DD_주제.md` |
| 위치 | `conversations/YYYY-MM/` |
| 필수 | YAML frontmatter, 요약, 결정사항 |

### 학습 내용 (learnings/)

| 항목 | 규칙 |
|------|------|
| 파일명 | `주제_분류.md` |
| 필수 | 핵심 개념, 적용 방법, 예시 |

### 치트시트 (cheatsheets/)

| 항목 | 규칙 |
|------|------|
| 파일명 | `주제_치트시트.md` |
| 특징 | 빠른 참조용, 표 중심 |

## 지식 활용 에이전트 (3개)

### 1. docs-searcher

| 항목 | 내용 |
|------|------|
| 역할 | 문서 검색 |
| 모델 | haiku (빠른 검색) |
| 입력 | 검색어, 범위, 기간 |
| 출력 | 관련 문서 목록 + 발췌문 |

**사용 예:**
```
@docs-searcher "컨텍스트 관리" 검색해줘
@docs-searcher scope=learnings 최근 7일 검색
```

### 2. knowledge-analyzer

| 항목 | 내용 |
|------|------|
| 역할 | 패턴/인사이트 분석 |
| 모델 | sonnet (분석력) |
| 입력 | 분석 주제, 범위 |
| 출력 | 패턴, 인사이트, 추천 |

**사용 예:**
```
@knowledge-analyzer 이번 달 반복되는 문제 분석해줘
@knowledge-analyzer 성공 사례에서 공통점 찾아줘
```

### 3. context-builder

| 항목 | 내용 |
|------|------|
| 역할 | 작업 시작 전 컨텍스트 구축 |
| 모델 | haiku (빠른 수집) |
| 입력 | 작업명, 키워드 |
| 출력 | 관련 히스토리, 주의사항, 추천 |

**사용 예:**
```
@context-builder "MCP 서버 개발" 시작 전 알아야 할 것 정리해줘
@context-builder "카카오 알림톡" 이전에 뭐 결정했었지?
```

## 지식 활용 스킬 (2개)

### knowledge-retrieval

- 통합 검색 스킬
- docs/, claude/, prompts/ 전체 검색
- 키워드/날짜/카테고리 필터

### weekly-insight

- 주간 인사이트 자동 생성
- 매주 월요일 실행
- docs/insights/YYYY-WW_주간인사이트.md 저장

---

## 지식 활용 흐름

```
새 작업 시작
      ↓
context-builder로 관련 히스토리 수집
      ↓
작업 수행
      ↓
결과/학습 내용 docs/에 저장
      ↓
weekly-insight로 주간 정리
      ↓
knowledge-analyzer로 패턴 분석
      ↓
다음 작업에 활용
```

---

## 저장 vs 활용

| 구분 | 저장만 | 저장 + 활용 |
|------|--------|-------------|
| 가치 | 낮음 | 높음 |
| 재사용 | 어려움 | 쉬움 |
| 학습 | 없음 | 지속적 |

**핵심:** 저장은 시작일 뿐, 활용이 진짜 목적!
