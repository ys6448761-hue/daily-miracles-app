# Part 1: 최수민님 방식 Q&A 분석

## 누구인가?

- 1인 유니콘 개발자
- 클로드 코드 기반 에이전틱 워크플로우 전문가
- 100개 서비스 런칭 목표

## 핵심 철학

> "AI는 추론, 반복은 코드로!"
> "내가 쓸 제품을 만들어라" (도그푸딩)

---

## Q&A 핵심 요약

### 피드백 루프 (Q1-3)

| 질문 | 답변 |
|------|------|
| 피드백은 어디에 기록? | 에이전트 .md에 직접 반영 |
| 별도 문서 필요? | X - 한 곳에 집중 |
| 언제 반영? | 실수 발견 즉시 지침에 추가 |

**핵심 원칙:**
- 별도 문서 X → 에이전트 .md에 직접 반영
- 실수 발견 → 지침 섹션에 즉시 추가
- "완벽한 설계보다 실행 후 수정"

### 스크립트화 판단 기준 (Q4-5)

**우선순위:**
1. 컨텍스트 소모량 (최우선)
2. 정확도/속도 필요성
3. 범용성 (MCP로 확장 가능성)

**스크립트화 대상:**
- 반복 작업 (3회 이상)
- 데이터 처리 (추출, 변환, 저장)
- 외부 API 호출

### 병렬 처리 (Q6-7)

```
에이전트가 서브 에이전트 호출
      ↓
배치 프롬프트: "10개씩 처리하고 반복"
      ↓
실패 시 전체 롤백 X, 실패 항목만 재시도
```

**실패 처리:**
- 전체 롤백 X
- 실패 항목만 로깅
- 재시도 큐에 추가

### 컨텍스트 관리 (Q8-9)

| 방식 | 설명 |
|------|------|
| 감지 | 컨텍스트 소진 시 경고 |
| **예방** | 스크립트화로 중간 데이터 안 남김 |

**핵심:** 감지보다 예방이 중요!

### 스킬 vs MCP (Q10-11)

| 구분 | 스킬 | MCP |
|------|------|-----|
| 범위 | 프로젝트 특화 | 범용적, 공유 가능 |
| 위치 | .claude/skills/ | mcp-servers/ |
| 재사용 | 프로젝트 내 | 전역 |

**MCP로 전환 시점:**
- 다른 프로젝트에서도 사용
- 외부 도구 연동 필요
- 독립 실행 필요

### 모델 선택 (Q12-13)

| 모델 | 용도 | 예시 |
|------|------|------|
| Opus | 복잡한 추론, 창의적 작업 | 전략 수립, 콘텐츠 기획 |
| Sonnet | 중간 난이도 | 분석, 코드 작성 |
| Haiku | 단순 반복 | 추출, 분류, 검색 |

## 적용 포인트

### 1. 프롬프트 아카이빙 습관

```
좋은 결과 나온 프롬프트
      ↓
변수화 ({{name}}, {{wish}})
      ↓
prompts/ 폴더에 저장
      ↓
사용량 10회 이상 → 에이전트 전환
```

### 2. 변수화 → 에이전트 전환

**전환 기준:**
- 사용 횟수 10회 이상
- 고정 패턴 확인
- 변수 3개 이상

### 3. 스크립트로 찌꺼기 방지

**Before:**
```
AI가 100개 항목 읽고 → 컨텍스트에 모두 저장 → 소진
```

**After:**
```
스크립트가 추출 → 파일로 저장 → AI는 결과만 참조
```

---

## 배운 점

1. **피드백은 분산시키지 마라** - 에이전트 .md 한 곳에 집중
2. **컨텍스트는 유한한 자원** - 예방이 최선
3. **실행 후 수정** - 완벽한 설계보다 빠른 실행
4. **도그푸딩 필수** - 만든 사람이 첫 번째 사용자
