# Part 6: 소원착지 전략

## 배경

소원이(사용자)별로 독립적인 프로젝트 환경이 필요하다는 전략적 결정.

---

## 핵심 개념

### 소원착지란?

```
소원이 등록
    ↓
12질문 응답
    ↓
기적지수 분석
    ↓
독립 프로젝트 생성 ← "소원착지"
    ↓
30일 여정 시작
```

**소원착지 = 소원이별 독립 프로젝트 생성 시점**

---

## 독립 프로젝트 구조

### 예상 폴더 구조

```
wishes/
├── sowoni_001/
│   ├── profile.json         # 소원이 프로필
│   ├── analysis.json         # 기적지수 분석 결과
│   ├── roadmap.pdf          # 30일 로드맵
│   ├── messages/            # 14개 응원 메시지
│   │   ├── day1_morning.md
│   │   ├── day1_evening.md
│   │   └── ...
│   ├── storybook/           # 스토리북 (선택)
│   └── logs/                # 발송 로그
├── sowoni_002/
└── ...
```

### 이점

| 항목 | 설명 |
|------|------|
| 독립성 | 소원이별 완전 분리 |
| 추적성 | 개별 히스토리 관리 |
| 확장성 | 새 콘텐츠 쉽게 추가 |
| 백업 | 개별 백업/복원 가능 |

---

## 필요한 스킬 (향후 개발)

### 1. agent-creator

| 항목 | 내용 |
|------|------|
| 역할 | 새 에이전트 자동 생성 |
| 입력 | 에이전트 유형, 설정 |
| 출력 | claude/agents/에 .md 파일 |

### 2. project-creator

| 항목 | 내용 |
|------|------|
| 역할 | 소원이별 독립 프로젝트 생성 |
| 입력 | 소원이 정보, 분석 결과 |
| 출력 | wishes/sowoni_XXX/ 폴더 구조 |

---

## 전체 흐름

```
소원이 등록 (웹/카카오)
        ↓
12질문 응답 수집
        ↓
miracle-analyzer 실행
        ↓
project-creator 실행 ← 소원착지!
        ↓
독립 프로젝트 생성
        ↓
roadmap-generator 실행
        ↓
wish-writer 실행
        ↓
message-dispatcher 스케줄링
        ↓
7일간 자동 발송
```

## 다음 단계 (P0)

### 1. agent-creator 스킬 개발

```markdown
입력:
- 에이전트 유형 (분석/콘텐츠/운영)
- 이름, 역할 설명
- 필요 도구

출력:
- claude/agents/{name}.md
- 필수 섹션 자동 포함 (역할, 지침, 자가검증, 피드백)
```

### 2. project-creator 스킬 개발

```markdown
입력:
- 소원이 ID

출력:
- wishes/sowoni_{id}/ 폴더 생성
- 하위 구조 자동 생성
- profile.json 초기화
```
