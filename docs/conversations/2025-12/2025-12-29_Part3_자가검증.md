# Part 3: 자가 검증 시스템

## 핵심 원칙

> "만든 사람이 첫 번째 사용자가 되어라" (도그푸딩)
> "스킬에 검증 로직도 포함하라"

---

## 자가 검증 4단계

### 1단계: 도그푸딩

```
개발 완료
    ↓
개발자가 직접 사용 (첫 번째 사용자)
    ↓
문제 발견
    ↓
즉시 수정
```

**체크리스트:**
- [ ] 내가 실제로 사용해봤는가?
- [ ] 불편한 점은 없었는가?
- [ ] 예상대로 동작했는가?

### 2단계: 검증 로직 내장

**스킬/에이전트에 검증도 포함:**

```markdown
## 자가 검증

작업 완료 후 다음을 스스로 확인하라:

1. [ ] 출력 형식이 맞는가?
2. [ ] 필수 필드가 모두 있는가?
3. [ ] 값이 유효 범위 내인가?
4. [ ] 예외 케이스를 처리했는가?
```

### 3단계: 지침 보강

```
오류 발견
    ↓
에이전트 .md 지침 섹션에 추가
    ↓
다음부터 자동 적용
```

**예시:**
```markdown
## 지침

> 피드백을 반영하여 계속 추가되는 작업 지침입니다.

1. 기적지수는 50-100 범위로 제한
2. 음수 값 발견 시 최소값(50)으로 대체
3. [2025-12-29 추가] 소수점 둘째자리까지만 표시
```

### 4단계: 파이프라인 종합 검증

```
개별 스킬 검증 완료
        ↓
파이프라인 전체 실행
        ↓
입력 → 출력 end-to-end 검증
        ↓
최종 승인
```

---

## 팀별 검증 체크리스트

### 코미 (기획/운영)

- [ ] 지시서가 명확한가?
- [ ] Phase별 산출물이 정의되어 있는가?
- [ ] 완료 기준이 명시되어 있는가?

### 재미 (콘텐츠)

- [ ] 톤앤매너가 일관적인가?
- [ ] 대상 연령에 적합한가?
- [ ] 브랜드 가이드를 준수했는가?

### 루미 (기술)

- [ ] 코드가 정상 실행되는가?
- [ ] 에러 핸들링이 있는가?
- [ ] 성능이 기준 내인가?

### 여의보주 (전략)

- [ ] 목표와 일치하는가?
- [ ] ROI가 긍정적인가?
- [ ] 리스크가 관리 가능한가?

### Claude Code (실행)

- [ ] 지시서 모든 Phase 완료했는가?
- [ ] 생성해야 할 파일이 모두 존재하는가?
- [ ] MASTER-INDEX가 업데이트되었는가?

---

## Bad Signal 감지

| 신호 | 의미 | 대응 |
|------|------|------|
| "잘 모르겠습니다" | 정보 부족 | 컨텍스트 보강 |
| 반복 오류 | 지침 누락 | 지침 추가 |
| 형식 불일치 | 템플릿 문제 | 템플릿 수정 |
| 누락된 필드 | 검증 실패 | 검증 로직 추가 |

---

## 검증 실패 시 행동

```
검증 실패
    ↓
문제점 기록 (피드백 기록 섹션)
    ↓
지침 보강 (지침 섹션)
    ↓
재작업
    ↓
재검증
    ↓
3회 실패 시 → 에스컬레이션
```

---

## 에이전트에 적용된 검증 섹션

모든 에이전트에 다음 섹션이 추가됨:

```markdown
---

## 자가 검증

작업 완료 후 다음을 스스로 확인하라:

1. [ ] [작업별 체크 1]
2. [ ] [작업별 체크 2]
3. [ ] [작업별 체크 3]

### 검증 실패 시
- 문제점 기록
- 재작업 실행
- 다시 검증
- 3회 실패 시 에스컬레이션

---

## 피드백 기록 (지침에 반영됨)

| 날짜 | 문제 | 해결 | 지침 추가 내용 |
|------|------|------|----------------|
| YYYY-MM-DD | 문제 설명 | 해결 방법 | 추가된 지침 |
```
