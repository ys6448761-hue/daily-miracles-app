# 대화 저장소 워크플로우

> Raw → 정본 승격 프로세스 가이드

---

## 1. 폴더 규칙

### Raw (원문/개인 기록)
```
docs/raw/conversations/YYYY-MM/
```
- 개인 메모, 채팅 원문, 회의록 원본
- 자유 형식 (템플릿 권장)
- 팀 공유 기준 아님

### 정본 (Single Source of Truth)
```
docs/conversations/YYYY-MM/
```
- 팀 공유용 최종 문서
- 반드시 템플릿 준수
- 결정/액션의 유일한 출처

---

## 2. 파일명 규칙

### Raw
```
YYYY-MM-DD_주제_작성자.md
예: 2025-12-30_ACT자동화_코미.md
```

### 정본
```
YYYY-MM-DD_주제.md
예: 2025-12-30_ACT자동화.md
```

---

## 3. 승격 책임자

| 역할 | 담당자 | 권한 |
|------|--------|------|
| 승격 담당 | 코미 (COO) | Raw → 정본 변환 |
| 최종 승인 | 푸르미르 (CEO) | 중요 결정 승인 |
| 도구 지원 | Code | 에이전트 실행 |

---

## 4. 승격 프로세스

### 표준 파이프라인

```
Raw 파일(들)
    ↓
[1] docs-searcher     → 관련 기존 문서/결정 링크
    ↓
[2] decision-extractor → 결정/액션 추출
    ↓
[3] summarizer        → 요약 생성
    ↓
[4] knowledge-analyzer → 인사이트/학습 포인트
    ↓
[5] (선택) feedback-analyzer → 사용자 피드백 처리
    ↓
정본 문서 1개
```

### 단계별 설명

| 순서 | 에이전트 | 입력 | 출력 |
|------|----------|------|------|
| 1 | docs-searcher | topic 키워드 | 관련 문서 링크 리스트 |
| 2 | decision-extractor | Raw 파일(들) | 결정/액션 표 |
| 3 | summarizer | Raw 파일(들) | 요약 (3~7줄) |
| 4 | knowledge-analyzer | Raw + 정본 | learnings/insights |
| 5 | feedback-analyzer | 피드백 포함 Raw | 피드백 정리 |

---

## 5. 정본 완료 조건 (Definition of Done)

정본 문서가 아래를 **모두** 만족해야 승격 완료:

- [ ] **요약** 3~7줄 존재
- [ ] **결정 사항** 섹션 존재 (없으면 "없음" 명시)
- [ ] **액션 아이템** 표 존재 (없으면 "없음" 명시)
- [ ] **참고**에 Raw 파일 경로 링크 1개 이상
- [ ] **파일명**이 규칙 준수 (YYYY-MM-DD_주제.md)

---

## 6. 운영 루틴

### 권장 주기
- **일일**: 중요 대화/회의 후 바로 승격
- **주간**: 금요일 EOD 정본 일괄 검토

### 체크리스트
1. Raw 폴더에 새 파일 확인
2. 승격 필요 여부 판단
3. 에이전트 파이프라인 실행
4. 정본 완료 조건 확인
5. MASTER-INDEX 업데이트

---

## 7. 팀 운영 가이드

> **"raw는 각자 자유롭게, docs는 팀 정본만. 결정/액션은 docs에만 남긴다."**

### 원칙
1. Raw는 원문 창고 - 자유롭게 기록
2. 정본은 유일한 진실 - 팀 공유 기준
3. 결정/액션은 정본에서만 검색

### 루미 활용
- 루미는 GitHub 직접 접근 불가
- 코미가 **정본 내용을 채팅에 복붙/파일 업로드**로 공유
- 루미가 정본 기반으로 리포트/다음 액션 분석
