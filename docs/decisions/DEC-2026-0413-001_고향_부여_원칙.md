# DEC: 고향 부여 원칙
> 파일: `DEC-2026-0413-001_고향_부여_원칙.md`
> 결정일: 2026-04-13 | 상태: CONFIRMED

---

## 결정 요약

고향은 자유롭게 생성할 수 없다. 인증된 파트너 장소에서의 QR 스캔으로만 부여된다.

---

## 세부 규칙

| 규칙 | 내용 |
|------|------|
| 첫 고향 | 여수 인증 파트너에서만 획득 |
| 고향 확장 | 이후에도 인증 지역 파트너에서만 추가 가능 |
| 고향 성격 | 선택이 아닌 '부여' — 사용자가 임의로 지정 불가 |
| 고향 변경 | 불가 (최초 부여 고향 고정) |
| 고향 수 | 별 1개당 고향 1곳 |

---

## 결정 이유

1. **브랜드 가치 유지** — 고향이 남발되면 희소성·감동 소멸
2. **확장 구조 통제** — 무분별한 파트너 확장 방지
3. **지역 기반 경제 모델 유지** — 여수 방문이라는 실제 행동이 전제되어야 의미 있음

---

## 기술 구현 기준

- `dt_stars.hometown_partner_id` — NULL이면 고향 없음
- `POST /api/hometown/arrive` — QR 스캔 시만 고향 등록 처리
- 관리자 임의 설정 금지 (스크립트의 샘플 별은 예외 — `is_sample=TRUE` 플래그로 구분)
