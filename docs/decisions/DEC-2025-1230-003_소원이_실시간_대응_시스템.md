## 중앙 통제실

**플랫폼:** Airtable

### Wishes Inbox 테이블 구조
| 필드 | 타입 | 설명 |
|------|------|------|
| wish_id | ID | 고유 식별자 |
| created_at | DateTime | 접수 시간 |
| channel | Select | 인입 채널 |
| status | Status | 상태 |
| priority | Select | 우선순위 |
| type | Select | 소원 유형 |
| assigned_to | Link | 담당자 |
| sla_deadline | DateTime | SLA 마감 |

## 처리 플로우

```
1. 소원이 접수
   ↓
2. Airtable Wishes Inbox (NEW)
   ↓
3. 자동 분류 + 배정 + SLA 타이머
   ↓
4. 알림 발송 (운영채널 + 담당자 DM)
   ↓
5. 초동응답 자동 발송 (ACK)
   ↓
6. 작업 진행 (IN_PROGRESS)
   ↓
7. 여의보주 검수 (APPROVED)
   ↓
8. 7일 여정 시작 (STARTED)
```

## 실행 계획

### Code 할 일
1. WishRouter 에이전트 구현
2. Airtable Wishes Inbox 테이블 연동
