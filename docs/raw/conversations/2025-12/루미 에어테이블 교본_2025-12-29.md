## 1) Airtable 베이스 확장: `Wishes Inbox` 테이블 스키마 (필드/타입 추천)

### ✅ 테이블: `Wishes Inbox`

**목표:** 소원이 유입의 “단일 상황판” + 신호등 라우팅 + SLA 관리

**필수 필드**

* `Wish ID` (Formula)

  * 예: `WISH-20251230-001` (Created time 기반 자동 생성)
* `Created Time` (Created time)
* `Source` (Single select)

  * Form / Kakao / Landing / DM / 기타
* `Contact Channel` (Single select)

  * Kakao / Email / Phone / Unknown
* `Contact` (Text) *(민감정보는 최소, 식별키 정도만)*
* `Wish Text (Raw)` (Long text) *(원문, 권한 제한 권장)*
* `Summary` (Long text) *(AI 요약본: 알림에 사용하는 건 이 필드)*
* `Category` (Single select)

  * 새출발/치유/관계/성장/도전/기타
* `Signal` (Single select)

  * 🟢 Green / 🟡 Yellow / 🔴 Red / ✨ VIP
* `Risk Flags` (Multiple select)

  * 개인정보/자해암시/폭력/법적/클레임/성적/아동/기타
* `Status` (Single select)

  * NEW → ACK → IN_PROGRESS → APPROVED → STARTED → DONE / HOLD / BLOCKED
* `Owner (Ops)` (Collaborator) — 기본: 코미
* `Owner (Data)` (Collaborator) — 루미(🟡 담당)
* `Owner (Creative)` (Collaborator) — 재미
* `Owner (Tech)` (Collaborator) — Code
* `Human Owner` (Collaborator) — 푸르미르/여의보주(🔴/VIP)
* `SLA: Ack Due` (Formula/Date)

  * NEW 생성 시점 + X분
* `SLA: Breach` (Formula checkbox)

  * `NOW() > {SLA: Ack Due}` AND Status=NEW
* `Thread Link` (URL) *(Slack/Discord/상담 스레드 링크)*
* `Decision Link` (Link to `Decisions`) *(필요 시)*

**선택(있으면 좋은)**

* `VIP Note (Human)` (Long text) — 여의보주 수기 메모
* `Outbound Message Draft` (Long text) — 초동응답 초안
* `Approved At / Started At` (Date)

## 2) 자동화 룰(신호등 + 상태 전이): Airtable Automations 6개로 MVP

### Automation #1: “NEW 생성 → 라우팅 시작”

* Trigger: `When record created` (Status=NEW)
* Actions:

  1. `Run script` or `Webhook`(WishRouter 호출)
  2. `Update record`: `SLA: Ack Due` 세팅(예: 30분)

### Automation #2: “🟢 Green → 자동 ACK 초안 + 채널 로그”

* Trigger: `Signal = 🟢`
* Actions:

  * `Post message` to Slack/Discord `#소원이-인입` (요약 카드)
  * `Update record`: `Outbound Message Draft` 생성(템플릿)

### Automation #3: “🟡 Yellow → 루미 알림 + 수동 분류 요청”

* Trigger: `Signal = 🟡`
* Actions:

  * 루미 DM + 운영채널 포스트(“수동 분류 필요”)
  * `Owner (Data)=루미` 자동 할당

### Automation #4: “🔴 Red → 발송 차단 + 사람 카톡 알림”

* Trigger: `Signal = 🔴`
* Actions:

  * `Status = BLOCKED`
  * 코미 + 여의보주 + 푸르미르 “긴급” 알림
  * (카톡은 Code가 메시징 API로 연결)

### Automation #5: “✨ VIP → 여의보주 수기 루프”

* Trigger: `Signal = ✨ VIP`
* Actions:

  * `Human Owner=여의보주`
  * 여의보주에게 요약 + 원문 링크 전달(원문은 권한 주의)

### Automation #6: “SLA breach → 에스컬레이션”

* Trigger: `SLA: Breach = true`
* Actions:

  * 담당자 재알림 → 코미 에스컬레이션(연속 2회면 🔴로 승격 옵션)

## 3) WishRouter 에이전트 스펙 (Code에게 바로 전달)

### 🎯 목적

`Wishes Inbox`에 NEW가 생기면
→ **요약 + 신호등 분류 + 담당 배정 + 알림 payload 생성**을 자동으로 수행

### 입력 (Webhook Payload)

```json
{
  "wish_id": "WISH-20251230-001",
  "created_time": "2025-12-30T10:12:00+09:00",
  "source": "Form",
  "contact_channel": "Kakao",
  "raw_text": "소원 내용 원문..."
}
```

### 처리 로직(권장: 룰 1차 + LLM 2차)

1. **룰 기반 1차 필터**

* 금칙/민감 키워드(자해/폭력/성적/아동/개인정보 패턴 등) 감지 → 🔴
* 너무 짧음/의미 불명/스팸 의심 → 🟡

2. **LLM 기반 2차 분류**

* `Summary(3~5줄)` 생성
* `Category` 분류
* `Signal` 최종 판정(🟢/🟡/🔴/✨)
* `Risk Flags` 태깅

3. **담당 배정**

* 🟢: 코미(Ops) 기본 + 재미(크리에이티브) + Code(발송)
* 🟡: 루미(데이터 검수) 우선 배정
* 🔴/VIP: Human Owner 추가 배정

### 출력 (Airtable Update + 알림용 카드)

```json
{
  "summary": "요약...",
  "category": "치유",
  "signal": "Green",
  "risk_flags": ["개인정보"],
  "owners": {
    "ops": "코미",
    "creative": "재미",
    "data": "루미",
    "tech": "Code",
    "human": "여의보주"
  },
  "notification_card": {
    "title": "NEW 소원이 도착",
    "fields": {
      "Wish ID": "WISH-20251230-001",
      "Signal": "🟢",
      "Category": "치유",
      "Ack Due": "30분 이내",
      "Summary": "..."
    }
  }
}
```

## 4) 초동응답 템플릿(자동 생성용) — Green 전용

**Outbound Message Draft (예시)**

* “소원이 접수 완료 🌟 지금 마음을 잘 받았어요.
  오늘 안에 ‘소원그림/문구’ 초안을 준비해서 안내드릴게요.
  혹시 꼭 포함되면 좋은 키워드(색/분위기/기억)가 있다면 한 줄로 남겨주세요.”

> 🟢는 “안심”이 핵심이라, **짧고 따뜻하게 + 다음 단계 예고**만 있어도 충분.

## 5) 코미가 운영에서 딱 기억할 3가지(실수 방지)

1. **원문은 알림에 싣지 말기** → 알림은 `Summary`만 (민감 리스크 최소화)
2. **SLA는 ‘ACK’만** 먼저 잡기 → 24시간 대응 강박 방지
3. **🔴는 무조건 BLOCKED로 멈추기** → 사람 판단 전 자동 발송 금지

---

원하면 내가 다음으로 **(1) 신호등 판정 규칙표(키워드/패턴) 초안**이랑 **(2) 카톡 긴급 알림 메시지 포맷**, **(3) Airtable View 구성(상황판 4개: NEW/노란불/빨간불/VIP)**까지 한 번에 뽑아서 코미가 그대로 세팅할 수 있게 만들어줄게.
