📝 Aurora5 팬덤 OS 구축 결정 회의록

일시: 온라인 비동기 (2026-01-04)

참석: 푸르미르님(CEO), 루미(데이터/전략), 코미(총괄)

1) 안건

Aurora5 팬덤을 “콘텐츠 중심”이 아닌 완주 중심 OS로 재설계

완주 2트랙(시즌/용궁) 확정 및 멘토·수익 구조 연결

멘토/팬덤 자동화 시스템(MVP) 범위 확정

2) 주요 논의 내용 (요약)

팬덤은 조회/팔로워가 아니라 완주를 늘리는 시스템이어야 한다는 데 합의

“완주”를 하나로 두면 혼란이 생기므로,

시즌완주소원이(NSM 직결)

용궁완주소원이(멘토/리더/수익 직결)
로 2트랙 분리 결정

용궁완주소원이는 “상위 레벨”이 아니라 검증된 멘토 자격증으로 정의

수익은 전면에 두지 않고, 신뢰 → 권한 → 결과 구조로 설계

NFC 용궁 카드는 명함/영업 도구가 아닌 신뢰 증표로 사용

멘토–새싹 매칭은 자동/반자동 혼합, 민감 신호는 즉시 HOLD 처리

멘토 품질은 개인 역량이 아니라 스크립트·가드레일·태그 시스템으로 표준화

3) 결정 사항 (DEC 확정)
✅ D1. 북극성 지표(NSM)

시즌완주율(7일) × 유료전환율

✅ D2. 역할 OS

새싹소원이 → 기록소원이 → 시즌완주소원이 → 전달소원이 → 채택소원이 → 수호소원이 → 길잡이소원이

레벨 개념 폐기, 정체성/역할 기반 운영

✅ D3. 완주 2트랙

A) 시즌완주소원이

이벤트: SEASON_COMPLETE

B) 용궁완주소원이

이벤트: YONGGUNG_WISH_COMPLETE

기본 상태: HOLD → 검증 후 PASS

✅ D4. 콘텐츠 원칙

모든 콘텐츠에 참여 슬롯 1개 강제(빈칸/선택/증언/전달/채굴)

✅ D5. 용궁완주 검증

Self Proof(리포트 10문항)

Peer Proof(동료 검증 체크리스트)

System Proof(이벤트/로그)

✅ D6. 용궁 카드(NFC)

배지 / 소원(측정) / 증언 / 변화 / 돕는 방식 / 연결 / 디스클레이머

가격·수익 UI 전면 배제

✅ D7. 멘토 운영

멘토 행동 규약 7대 원칙 적용

상황 태그(T01~T08) 기반 스크립트 자동 추천

민감 키워드 감지 시 즉시 HOLD + 라우팅

✅ D8. MVP 구현 범위

Week1~Week4 단계별 구현 스펙 확정

완주 2트랙, 역할/권한, 매칭, 검증, 가드레일 우선
