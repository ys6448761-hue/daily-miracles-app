└─────────────────────────────────────────────────────┘
5) 주문 상태 머신 (State Machine)
CREATED → PAID → QUEUED → GENERATING → GATED → STORING → DELIVERING → DONE

실패 상태 (단계별):
- FAIL_PAYMENT_VERIFY
- FAIL_GENERATION
- FAIL_GATE
- FAIL_STORAGE
- FAIL_DELIVERY
- FAIL_BUDGET (비용 초과)
- SECURITY_FAIL (웹훅 검증 실패)

환불 상태:
- REFUND_REQUESTED → REFUNDED

9) 재시도/실패 처리
9.1 재시도 정책

Delivery 재시도: 채널별 2회
- 이메일 실패 → 카톡 폴백 가능

최종 실패 시:
- NEEDS_HUMAN 큐로 이동
- 운영 알림 발송
- 자동 안내 메일 발송 (사과 + CS 연락처)
9.2 비용 상한
Starter:
- 토큰: 최대 10,000
- 이미지: 최대 5

Plus:
- 토큰: 최대 15,000
- 이미지: 최대 12

Premium:
- 토큰: 최대 25,000
- 이미지: 최대 12
- 문서: 최대 2

예산 초과 시:
- "저비용 모드"로 전환 또는
- FAIL_BUDGET + 운영 플래그
10) 크레딧(커스터마이즈) 워크플로우

11) 테스트 계획 (DoD)
11.1 E2E 필수 통과 케이스
✅ Case 1: Starter 결제 → 스토리북 생성 → 이메일 전달 OK
✅ Case 2: Plus 결제 → 스토리북+웹툰 생성 → 전달 OK
✅ Case 3: Premium 결제 → 전부 생성 → 전달 OK
11.2 운영 필수 케이스
✅ Case 4: 웹훅 중복 수신 → 결과 1회만 전달
✅ Case 5: 생성 실패 → 재시도 후 FAIL 기록 + 운영큐 이동
✅ Case 6: 전달 실패(이메일) → 재시도 + 카톡 폴백
✅ Case 7: Gate FAIL → 격리 + 미전달 + 운영 알림
✅ Case 8: 환불(결제 후 즉시) → 생성 중단 + 상태 반영
✅ Case 9: 비용 초과 → FAIL_BUDGET + 저비용 모드 또는 중단
11.3 품질 기준
- Ethics Score >= 13/16
- gate_result != FAIL
- 상태는 반드시 DONE 또는 FAIL_*로 종결 ("중간 멈춤" 금지)
- PDF 파일 생성 성공
- 링크 전달 성공 (이메일/카톡 중 1개라도)
12) 운영 대시보드 (최소 화면)
