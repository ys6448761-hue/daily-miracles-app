## 2) 마케팅 오케스트레이터 엔진 설계 (핵심 개념)

### ✅ 목표

콘텐츠 생산이 아니라 **목표 달성형 운영 시스템**

* Goal(목표/KPI) → Exec(제작/배포) → Measure(측정) → Learn(학습/지식화)

### ✅ 구성(1 오케스트레이터 + 서브 에이전트)

* **Marketing Orchestrator(MO)**: 주간 목표 달성 책임
* 서브 에이전트:

  * Insight Analyst(병목/실험)
  * Offer/Copy Agent(카피/오퍼)
  * Script Agent(숏츠/릴스 대본)
  * Design Prompt Agent(카드/썸네일 구조)
  * Distribution Agent(배포/예약/댓글/DM)
  * QA/Gate Agent(리스크 검수)

## 3) 완전 자동화를 위한 기반 2개 확정

### ✅ (A) Content Atom Schema v1.0 (원천 콘텐츠 구조)

콘텐츠를 **원자 단위로 표준화**해서 채널별로 자동 변환하도록 설계

* Meta / Hook / Problem / Insight / Action / CTA / Guardrail
* UTM/퍼널 이벤트 추적 포함
* 승자 패턴 자동 축적(WINNING_ATOMS)

### ✅ (B) Marketing QA/Gate Standard v1.0 (안전장치)

완전 자동 배포를 위해 **PASS/WARN/HOLD/FAIL** 4단계 게이트 확정

* FAIL: 정책/안전 위반(격리)
* HOLD: 민감/단정/조작 가능성 → 사람 승인(HITL)
* WARN: 자동 완화 후 게시 가능
* PASS: 자동 게시

추가 규칙

* 디스클레이머 자동 삽입 조건
* 중복/피로도 제어(쿨다운)
* HOLD/FAIL만 알림

- voice_access
