## 1️⃣ P0: 홈페이지 수정 (오늘)

**목표:** "결제/문의/예약"으로 이어지는 **단일 CTA 링크**

### 수정 항목 (dailymiracles.kr / Wix)

| 항목 | 내용 |
|------|------|
| 상단 CTA 버튼 | `지금 예약/신청하기` (1개만!) |
| 가격/정책 요약 | 요트(평일/불꽃) / 케이블카 / 체험(주말 장소) |
| FAQ | 3개만 |

### FAQ 3개

```
Q1. 주말/공휴 체험 장소는?
→ 갤러리/몽돌해변 고정 (사무실 앞 1~2분)

Q2. 불꽃 요트 요금은 언제 적용?
→ 금/토/일/공휴

Q3. 결제/확정 방식은?
→ 예약 접수 → 확정 안내
```

### 링크 구조

```
홈페이지 CTA → 앱 예약 폼 (UTM 포함 권장)
상담 채널(카카오) → 법인폰 연결 후 "상담하기" 노출
```

### ✅ 산출물
- [ ] 수정 완료된 랜딩 URL 1개

## 2️⃣ P0: 카카오 법인폰 연결 + 테스트 (내일)

**목표:** "문의 유입 → 자동응답 → 담당자 알림 → 접수 기록" 끊김 없이

### 테스트 시나리오 5개 (각 1분)

| # | 시나리오 | 기대 결과 |
|---|----------|-----------|
| 1 | "요트 예약 문의" | 자동응답 OK |
| 2 | "불꽃 요트 가격" | 안내 템플릿 OK |
| 3 | "체험 주말 장소" | 갤러리/몽돌 고정 안내 |
| 4 | "결제 방법" | 결제/확정 플로우 안내 |
| 5 | 욕설/민감 키워드 | 필터/에스컬레이션 동작 |

- [ ] 테스트 스크린샷 5개

## 3️⃣ P0: 운영 자동화 (가격 반영)

**목표:** "상품 선택 → 날짜 → variant → sell/cost 자동 결정" + "주말 체험 장소 강제"

### Code 지시 핵심

```
1. 가격/정책 패치 v1.1 반영 ✅
2. variant(priceTier) 필드 도입 (요트/유람선)
3. experience 주말/공휴 location 강제 로직 (P0!)
```

### ✅ 산출물
- [ ] API 응답에 `priceVersion` 확인
- [ ] API 응답에 `variant` 확인
- [ ] API 응답에 `dayType/weekType` 확인

## 4️⃣ P1: 마케팅 자동화 (오픈 즉시)

**목표:** 최소 자동화로 "유입 → 리드 → 후속 메시지 → 리마케팅" 루프

### 필수 이벤트 4개

```
1. landing_view      (랜딩 페이지 조회)
2. cta_click         (CTA 버튼 클릭)
3. lead_submit       (예약/문의/폼 제출)
4. purchase_or_confirm (결제/확정)
```

### 자동 메시지 (2단계만)

| 타이밍 | 내용 |
|--------|------|
| 즉시 | "접수 완료 + 안내" |
| D+1 | "가격/일정/장소 핵심 요약 + 재문의 버튼" |

### ✅ 산출물
- [ ] 이벤트가 찍히는 위치 확인 (대시보드 or 로그)

## 5️⃣ P2: 웹툰 & 기적영상 자동화 (루미 담당)

**루미 담당:** 입력 → 처리 → 출력 파이프라인 설계  
**Code/재미 담당:** 실제 구현(코드) / 디자인

### A) 입력 (소스 데이터)

```
- 소원/후기/스토리 소재
- 키워드
- 감정톤
- 해피엔딩 포인트
- 개인정보/실명 제거 규칙 (안전 필수!)
```

### B) 처리 (자동 생성 단계)

```
【6컷 스토리 시놉시스】
컷1: 문제/갈등
컷2: 작은 신호
컷3: 선택/전환
컷4: 도움/동행
컷5: 변화
컷6: 기적/완주

【추가 생성물】
- 대사/나레이션 분리
- 영상용 내레이션 스크립트 30~45초
- 썸네일 카피 3종
```

### C) 출력 (콘텐츠 패키지)

```
【웹툰】
- 컷별 텍스트 (장면설명 + 대사)
- 해시태그 세트

【영상】
- 나레이션 스크립트
- 자막본
- 컷 구성표 (스토리보드 텍스트)
```
