# 1) Notion DB ①: Offline Ops Log (운영대장) — 필드 타입 표준

## ✅ 필드(이름 / 타입 / 옵션)

* **요청 제목** / `Title`
* **접수일시** / `Date` (시간 포함)
* **채널** / `Select`

  * `카톡` `문자` `전화` `이메일` `Wix` `앱`
* **이름** / `Text`
* **연락처** / `Text`
* **유형** / `Select`

  * `입항` `견적` `결제` `환불` `일정변경` `기술오류` `기타`
* **긴급도** / `Select`

  * `상` `중` `하`
* **상품코드** / `Select`

  * `3pass` `crew` `exchange` `online` `기타`
* **요청 요약** / `Text`
* **상태** / `Select`

  * `new` `replied` `pending` `closed`
* **담당** / `People`
* **마지막 응답시간** / `Date` (시간 포함)
* **SLA(응답기한)** / `Date` (시간 포함)
* **증빙/링크** / `URL`
* **메모** / `Text` (또는 `Rich text`)

---

## ✅ 운영 룰(필수)

* 신규 접수 즉시 **상태=new**
* 1차 답변 발송 시 **replied**
* 추가 확인/입금대기/일정조율 등 진행 중이면 **pending**
* 완전 종료 시 **closed**
* 긴급도:

  * `상`: 결제/환불/당일 일정/긴급 민원
  * `중`: 일반 견적/일정 조율
  * `하`: 피드백/제휴/기타

---

## ✅ SLA 룰(초보자용, 자동이 어려우면 수동으로 고정)

* 기본: **접수일시 + 24시간**을 SLA로 입력
* 긴급(상): **접수일시 + 3시간** 권장

> 자동 계산(Formula)은 노션 설정 난이도가 있어 “우선 수동 입력”으로 시작하고, 안정화 후 자동화로 전환.

---

## ✅ 뷰(필수 6개) — 이름/필터 표준

### 1) 오늘 신규 (new)

* 필터:

  * `접수일시` = Today
  * `상태` = new

### 2) 미응답 전체 (new+pending)

* 필터:

  * `상태` is `new` OR `pending`

### 3) 긴급(상)

* 필터:

  * `긴급도` = 상
  * `상태` is not closed

### 4) 결제/환불

* 필터:

  * `유형` is `결제` OR `환불`

### 5) SLA 임박(3시간 이내)

* 필터:

  * `상태` is not closed
  * `SLA(응답기한)` is within next `3 hours` *(노션 UI에서 “다음 3시간” 선택)*

### 6) 완료(Closed)

* 필터:

  * `상태` = closed

# 3) Runbook(고정 문서) — 목차 표준(복붙)

## ✅ 비상 대응 Runbook v1.0

### 1) 비상 모드 ON(5분 전환)

* 공지 발송 채널: 카톡채널/문자/인스타 스토리
* 접수 채널 전환: 카톡 + 문자 + 이메일
* 운영대장 기록 시작
* 긴급 트리아지: 결제/환불 > 당일 일정 > 일반 문의

### 2) 고객 공지 템플릿(짧은/안심)

* 짧은(카톡/문자)
* 안심(페이지/고정공지)

### 3) Plan B 결제(계좌이체) 템플릿

* 알림톡 150자 버전
* 페이지 상세 버전

### 4) 담당자 역할표

* 코미: 공지/트리아지/진행
* 루미: 운영대장/SLA/리스크 감시
* 재미: 공지 이미지/스토리
* 여의보주: 톤/신뢰 문장
* 코드: 복구/원인/재발방지

### 5) 체크리스트(매일)

* 미응답 0건 목표
* SLA 임박 우선 처리
* 일일보고 생성 확인

# 4) Make 시나리오 ① — Daily Ops Update 자동 생성 (매일 10:00)

## ✅ 모듈 순서(입력 → 처리 → 출력)

1. **Scheduler**: 매일 10:00(Asia/Seoul)
2. **Notion: Search Objects** (Offline Ops Log)

   * 조건: `접수일시 = today`
   * 결과 건수 = `총 접수`
3. **Notion: Search Objects** (Offline Ops Log)

   * 조건: `접수일시=today AND 긴급도=상`
   * 결과 건수 = `긴급(상)`
4. **Notion: Search Objects** (Offline Ops Log)

   * 조건: `상태 in (new, pending)`
   * 결과 건수 = `미응답`
5. **(선택) HTTP: Make a request**

   * 서버 헬스체크 URL 1개(있으면)
6. **Router/Rules (상태 판단)**

   * 기본 룰:

     * 미응답 ≥ 10 → `주의`
     * 헬스체크 실패 → `비상`
     * 그 외 → `정상`
7. **Notion: Create Object** (Daily Ops Update)

   * 날짜=오늘, 숫자 3개 입력, 상태/알림톡/결제상태 선택
   * 본문 템플릿 자동 삽입
8. **(선택) Slack/Email/카톡 알림**

   * 코미에게 노션 링크 + 요약 3줄

---

# 5) Make 시나리오 ② — 비상 감지 알림(옵션)

## ✅ 트리거(택1)

* 헬스체크 실패가 연속 2회(5분 간격)
* 또는 미응답 건수(new+pending)가 15 이상

## ✅ 출력

* 코미/푸르미르에게 “비상 모드 ON” 알림 + Runbook 링크 + 공지 템플릿 포함
