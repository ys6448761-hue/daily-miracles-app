좋아. 아래는 **MO(마케팅 오케스트레이터)가 매주 자동으로 “주간 대시보드”를 표까지 채워서 출력**하도록 만든 **단일 프롬프트(복붙용)**이야.

핵심은: **입력(원자료) → 계산 규칙 → 표 자동 채움 → 결론 1줄 → 다음 주 3액션** 순서로 강제하는 것.

NSM = 시즌완주율(7일) × 유료전환율(Trial→Paid)
조회수/팔로워는 참고 지표이며, 결론/의사결정은 NSM 영향으로만 한다.

## 2) 계산 규칙(자동)
- 시즌완주율(7일) = season_complete / signups_or_starts
- 유료전환율 = paid_converted / trial_start
- NSM = 시즌완주율 × 유료전환율
- 참여슬롯 응답률(타입별) = responses / posts (posts가 0이면 0으로 처리)
- WoW 변화(%) = (this - prev) / prev × 100 (prev가 0이면 `NA`)
- 병목 판단: 퍼널 단계별 전환율을 계산해 가장 낮은 구간 1개를 병목으로 지정한다.
  - 전환율 예시:
    - signup_rate = signups_or_starts / visits
    - slot_rate = participation_responses / signups_or_starts
    - d1_rate = day1_active / participation_responses
    - d3_rate = day3_retained / day1_active
    - season_complete_rate = season_complete / signups_or_starts
    - trial_rate = trial_start / season_complete
    - paid_rate = paid_converted / trial_start

### 사용법(아주 짧게)

- 매주 원자료를 위 JSON으로 채워서 MO에게 붙여넣기 → **자동으로 표 계산/채움**
- JSON이 없으면 MO가 “필요 데이터 목록”만 뱉고 멈추게 설계됨

원하면 내가 **“입력 JSON을 자동으로 만드는 이벤트→지표 매핑표(로그에서 어떻게 집계할지)”**도 한 장으로 만들어줄게.
