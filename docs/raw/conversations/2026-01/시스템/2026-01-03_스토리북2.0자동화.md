# 2) 산출물 스펙 (티어별)

## 2.1 Starter 산출물

* storybook_{userId}_{date}.pdf (A4 세로 11p)
* storybook_{userId}_{date}.json (메타: promptsUsed, ethicsScore, generationTime)

## 2.2 Plus 산출물

* Starter 전부 +
* webtoon_{userId}_{date}.pdf 또는 webtoon_{userId}_{date}.png(zip)
  **정의(고정):** “새 장면 생성”이 아니라
  **스토리북 이미지 기반** *리사이즈/컷 분할/세로 카드화* (800×1280 등)
* 커스터마이즈 권리(크레딧) 포함(숫자 고정):

  * 이미지/컷 재생성 **3회**
  * 문장 톤/오타 수정 **1회**

## 2.3 Premium 산출물

* Plus 전부 +
* decision_map_{userId}_{date}.pdf (1~2p)

  * 선택지/전제조건/리스크/되돌림 기준/다음 확인 질문
* roadmap_90d_{userId}_{date}.pdf (개인화 강화판)
* 크레딧:

  * 이미지/컷 재생성 **8회**
  * 구조 리라이트(심화 문서 리라이트) **1회**

# 3) E2E 파이프라인 (필수 흐름)

**Payment Success Event**
→ user data fetch
→ generate by tier
→ Ethics Linter + Soft Fix + CI Gate
→ store outputs
→ deliver (email/kakao)
→ write back to DB/Airtable (status DONE, links)

---

# 4) 데이터/DB 스키마 최소 추가 (권장)

결제/생성 추적을 위해 최소 필드:

* tier: STARTER | PLUS | PREMIUM
* payment_id
* status: PAID → GENERATING → DELIVERED → DONE / FAIL
* ethics_score
* generation_time_sec
* assets: { storybookUrl, webtoonUrl, decisionMapUrl, roadmapUrl }
* credits_remaining: { regen_images, edit_text, rewrite_doc }

---

# 5) 윤리/운영 가드레일(강제)

* 모든 LLM 호출: Master Header prepend
* 모든 저장 전: Ethics Linter 실행
* FAIL: quarantine 격리 + 사용자 전달 금지
* WARN: 배너 삽입 + 전달 가능(운영 정책 유지)
* Webtoon은 “재구성”이므로 **새 감정 조작/예언 문구 금지**

# 6) 테스트 계획 (DoD)

### E2E 필수 통과 3케이스

1. Starter 결제 → 스토리북 생성/전달 OK
2. Plus 결제 → 스토리북+웹툰 생성/전달 OK
3. Premium 결제 → 전부 생성/전달 OK

### 품질 기준(각 케이스 공통)

* Ethics Score ≥ 13/16
* PDF 파일 생성 성공
* 링크 전달 성공(이메일/카톡 중 1개라도)
* 생성 실패 시 자동 롤백/상태 FAIL 기록

1) 결제/주문 이벤트 설계 보강 (중복·위변조·환불까지)

1.2 주문 상태 머신(권장: 더 촘촘하게)
CREATED → PAID → QUEUED → GENERATING → GATED → STORING → DELIVERING → DONE

실패는 단계별로 별도:

FAIL_PAYMENT_VERIFY

FAIL_GENERATION

FAIL_GATE

FAIL_STORAGE

FAIL_DELIVERY

REFUND_REQUESTED → REFUNDED 상태 추가

환불 시:

자동으로 바꾸면 안 되는 것: 사용자 사실관계, 관계/건강/법률의 결정적 조언

3.2 “리라이트 가능한 범위” 고정(운영 가드)
가능: 구조 정리, 우선순위 재배치, 문장 명료화, 행동 쪼개기

불가: 건강/법률/재정 단정 조언, 특정인 진단/평가, 예언/확정적 보장

4) 게이트 포인트(문서형 자산에 특화된 추가 체크)
기존 Ethics Score 외에, 문서형은 아래가 추가로 필요합니다.

Actionability Score(0~5): “바로 할 수 있는 행동이 있는가?”

Specificity Score(0~5): “측정 가능/기한/조건이 있는가?”

Overclaim Check: “반드시/무조건/확실히” 같은 단정 제거

→ PASS 조건 예시:

ethics_score ≥ 13/16

actionability ≥ 3

specificity ≥ 3

overclaim = false
