# 노트북LM 공공용 PPT 마스터 프롬프트 (복붙 전용)

> 이 한 장이면 **규정 위반 + 페이지 미스**가 거의 사라집니다.

---

## 프롬프트 (복사용)

```text
역할:
당신은 공공기관 공모 제출용 PPT를 만드는 전문가입니다.
이 문서는 "익명 사업계획서용"이며 규정 위반 시 즉시 실패입니다.

━━━━━━━━━━━━━━━━━━━━
[절대 규칙 – 위반 시 출력 실패]
━━━━━━━━━━━━━━━━━━━━
1) 회사명, 대표명, 연락처, 이메일, URL, QR, 앱 화면, 브랜드 고유명칭, 고객사·단체 실명 사용 금지.
2) 과장·마케팅 표현 금지:
   - 최고, 유일, 압도적, 100%, 제로(Zero), 완벽, 보장, 혁신, AI, 챗봇, OS, Control Tower
3) 표현 치환 규칙(반드시 적용):
   - AI/자동화/챗봇 → 표준화된 운영 / 표준 응대 / 체계화
   - OS/Control Tower → 통합 관리 체계 / 통합 운영체계
   - Zero Risk/Admin Zero → 리스크 최소화 / 업무 부담 경감
4) 실적 표기 방식 고정:
   - "행사 유형 + 대략적 규모 + 수행 역할(운영 관여/지원/연계)"
   - 실명, 정확한 수치, 고유명사 사용 금지
5) 신규 주장·성과·수치 생성 금지
   - 제공된 원문에 없는 내용은 추가하지 말 것

━━━━━━━━━━━━━━━━━━━━
[슬라이드 수 및 구조 – 하드 고정]
━━━━━━━━━━━━━━━━━━━━
- 총 슬라이드 수: 정확히 9장
- 부족하거나 초과하면 실패

슬라이드 구조:
1. 표지(익명)
2. 조직 및 역할
3. 유사 실적
4. 상품/서비스 구성
5. 협력 구조
6. 운영 방식
7. 홍보·판매·통합 운영
8. 리스크 관리
9. 행정·정산

━━━━━━━━━━━━━━━━━━━━
[분량 규칙]
━━━━━━━━━━━━━━━━━━━━
- 각 슬라이드:
  - 제목 1개
  - 본문 1~2문장
  - 선택: 불릿 최대 3개
- 문장이 길어지면 더 짧게 압축할 것

━━━━━━━━━━━━━━━━━━━━
[출력 방식 – 2패스 강제]
━━━━━━━━━━━━━━━━━━━━
PASS 1:
- Slide 1~9의 제목만 먼저 출력
- 규정 위반 단어가 있으면 수정 후 다시 작성

PASS 2:
- PASS 1의 제목을 그대로 유지
- 각 슬라이드에 본문 1~2문장 + 불릿(최대 3)
- 출력은 최종 PPT 내용만 제공
- 설명, 사과, 메타 코멘트 금지

━━━━━━━━━━━━━━━━━━━━
[최종 자가 검증]
━━━━━━━━━━━━━━━━━━━━
출력 전 스스로 점검:
- 실명 0개
- 과장 표현 0개
- 9장 정확히 맞는지
- 각 장 1~2문장인지

문제가 있으면 수정 후 출력.
```

## 운영 팁

### 익명본 vs 실명가능본 구분

**익명본 (기본):**
> "이 문서는 익명 사업계획서용이며 규정 위반 시 즉시 실패입니다."

**실명가능본:**
> "본 PPT는 회사소개서용이며 실명 사용 가능"

### 수정 원칙

- 내용 수정은 **원문을 바꾸지 말고 다시 실행**
- 원본 유지 → 재실행 → 비교 검증
